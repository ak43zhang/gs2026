# 主力净额计算逻辑对比分析

## 您的期望逻辑

```
时间1: 主力净额=0, 累计主力净额=0
时间2: 主力净额=0, 累计主力净额=0+0=0
时间3: 主力净额=0, 累计主力净额=0+0=0
...
时间10: 主力净额=0, 累计主力净额=0
时间11: 主力净额=20万, 累计主力净额=0+20万=20万
时间12: 主力净额=0, 累计主力净额=20万+0=20万
时间13: 主力净额=0, 累计主力净额=20万+0=20万
...
时间16: 主力净额=0, 累计主力净额=20万
```

**核心要求**:
1. 所有股票都参与计算（包括被门槛过滤的）
2. 被过滤的股票：主力净额=0
3. 累计主力净额 = 上一时刻累计值 + 当前主力净额（无论是否为0）
4. 累计值只增不减（除非主力净额为负）

## 当前代码逻辑分析

### 当前逻辑流程

```python
# 1. 初始化（所有股票）
df_now['main_net_amount'] = 0.0
df_now['main_behavior'] = '无主力'
df_now['main_confidence'] = 0.0
df_now['cumulative_main_net'] = 0.0

# 2. inner join（部分股票丢失）
merged = pd.merge(df_now, df_prev_main, ..., how='inner')
# 结果：~4800只股票（丢失200只）

# 3. 门槛过滤（大部分股票被过滤）
mask = (delta_amount >= 300000) & (delta_volume >= 20000)
valid_data = merged[mask].copy()
# 结果：~200只股票

# 4. 计算主力净额（仅这200只）
valid_data['main_net_amount'] = ...

# 5. 合并回df_now（how='left'）
df_now = df_now.merge(valid_data[...], on='stock_code', how='left')
df_now['main_net_amount'] = df_now['main_net_amount'].fillna(0)
# 结果：4800只股票
# - 200只：有计算值
# - 4600只：fillna(0) → 0

# 6. 计算累计主力净额
if 'cumulative_main_net' in df_prev_main.columns:
    prev_cumulative = df_prev_main[df_prev_main['cumulative_main_net'] != 0]
    # 问题：只筛选 cumulative_main_net != 0 的股票
    # 如果某股票上一时刻累计值为0，就被排除了！
    
    if not prev_cumulative.empty:
        df_now = df_now.merge(prev_cumulative, ..., how='left')
        df_now['cumulative_main_net'] = df_now['cumulative_main_net_prev'] + df_now['main_net_amount']
```

### 当前逻辑问题

**问题1：inner join丢失股票**
- df_now有5000只，df_prev_main有4800只
- inner join后只剩4800只
- **200只股票完全没有计算**

**问题2：门槛过滤**
- 4800只中，4600只被过滤
- 这些股票：main_net_amount=0（通过fillna）
- 但计算逻辑上是被"过滤"而非"计算为0"

**问题3：累计值筛选条件错误**
```python
prev_cumulative = df_prev_main[df_prev_main['cumulative_main_net'] != 0]
# 问题：如果某股票上一时刻累计值为0，就被排除
# 导致该股票累计值不会累加，永远为0
```

**问题4：累计值计算范围**
- 只合并了 `cumulative_main_net != 0` 的股票
- 如果某股票前10个时间累计都是0，第11个时间有值
- 第11个时间的累计值 = 0 + 20万 = 20万（正确）
- 第12个时间的累计值：因为筛选 `!= 0`，该股票被排除
- 第12个时间的累计值 = 0（错误！应该是20万）

**根本原因**：第12个时间，因为 `cumulative_main_net != 0` 筛选，该股票被排除，累计值重置为0。

## 修复方案

### 问题定位

**核心问题**：累计值计算时，只筛选 `cumulative_main_net != 0` 的股票，导致累计值断档。

### 修复方案A：删除筛选条件（推荐）

**修改**:
```python
# 修复前
prev_cumulative = df_prev_main[df_prev_main['cumulative_main_net'] != 0][['stock_code', 'cumulative_main_net']].copy()

# 修复后
prev_cumulative = df_prev_main[['stock_code', 'cumulative_main_net']].copy()
```

**效果**:
- 所有股票都参与累计值计算
- 即使上一时刻累计值为0，也会合并
- 当前主力净额=0时，累计值保持不变

**优点**:
- 改动最小
- 逻辑正确
- 符合期望

**缺点**:
- 无

---

### 修复方案B：使用left join保留所有股票

**修改1**: inner join改为left join
```python
# 修复前
merged = pd.merge(..., how='inner')

# 修复后
merged = pd.merge(..., how='left')
```

**修改2**: 删除门槛或降低门槛
```python
# 删除门槛过滤，或改为极低门槛
valid_data = merged.copy()
```

**修改3**: 累计值不筛选
```python
prev_cumulative = df_prev_main[['stock_code', 'cumulative_main_net']].copy()
```

**效果**:
- 所有5000只股票都参与计算
- 被过滤股票：主力净额=0
- 累计值正确累加

**优点**:
- 最彻底
- 100%覆盖

**缺点**:
- 改动较大
- 需要处理left join后的缺失值

### 修复方案C：修改累计值初始化逻辑

**修改**:
```python
# 修复前
df_now['cumulative_main_net'] = 0.0

# 修复后：从df_prev_main继承上一时刻累计值
df_now = df_now.merge(
    df_prev_main[['stock_code', 'cumulative_main_net']].rename(columns={'cumulative_main_net': 'cumulative_main_net_prev'}),
    on='stock_code',
    how='left'
)
df_now['cumulative_main_net_prev'] = df_now['cumulative_main_net_prev'].fillna(0)
df_now['cumulative_main_net'] = df_now['cumulative_main_net_prev'] + df_now['main_net_amount']
```

**效果**:
- 每只股票的累计值都从上一时刻继承
- 即使上一时刻为0，也会正确继承

**优点**:
- 逻辑清晰
- 累计值不会断档

**缺点**:
- 改动较大
- 需要重构累计值计算逻辑
