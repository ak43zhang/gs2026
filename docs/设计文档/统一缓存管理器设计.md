## 1. 背景与问题

### 1.1 当前问题

| 问题 | 现状 | 影响 |
|------|------|------|
| 代码分散 | 缓存预热逻辑分散在 app.py、各业务模块 | 难以维护 |
| 缺乏统一 | 各缓存自行实现预热逻辑 | 重复代码 |
| 扩展困难 | 新增缓存需修改 app.py | 容易出错 |
| 配置硬编码 | 同步/异步、优先级等无法配置 | 不够灵活 |

### 1.2 优化目标

- **统一入口**: 所有缓存通过统一接口管理
- **配置化**: 支持优先级、执行模式、超时等配置
- **可扩展**: 新增缓存只需注册，无需修改核心代码
- **可观测**: 统一预热结果报告

### 2.1 整体架构

```

│                        app.py                               │

│  │  from cache import init_all_caches, cache_manager   │   │

└──────────────────────────┬──────────────────────────────────┘
                           │
┌──────────────────────────▼──────────────────────────────────┐
│                    CacheManager (单例)                       │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  - _caches: Dict[str, CacheConfig]                  │   │
│  │  - register(config): 注册缓存                        │   │
│  │  - warmup_all(): 预热所有缓存                        │   │
│  │  - get_status(): 获取预热状态                        │   │
│  └─────────────────────────────────────────────────────┘   │
└──────────────────────────┬──────────────────────────────────┘
                           │
        ┌──────────────────┼──────────────────┐
        │                  │                  │
┌───────▼──────┐  ┌────────▼───────┐  ┌──────▼──────┐
│ 红名单缓存    │  │ 股票债券映射   │  │ 行业股票计数 │
│ (同步/关键)  │  │ (异步/高优)   │  │ (异步/普通)  │
└──────────────┘  └────────────────┘  └─────────────┘
```

### 2.2 核心类设计

#### CacheConfig (缓存配置)

```python
@dataclass
class CacheConfig:
    name: str                      # 缓存名称
    warmup_func: Callable          # 预热函数
    mode: WarmupMode = ASYNC       # 执行模式
    priority: CachePriority = NORMAL  # 优先级
    timeout: int = 60              # 超时时间
    retry: int = 1                 # 重试次数
```

#### WarmupMode (执行模式)

| 模式 | 说明 | 使用场景 |
|------|------|----------|
| SYNC | 同步执行，阻塞启动 | 关键缓存（如红名单） |
| ASYNC | 异步执行，后台线程 | 非关键缓存（如映射表） |

#### CachePriority (优先级)

| 优先级 | 值 | 说明 |
|--------|-----|------|
| CRITICAL | 1 | 关键，必须成功 |
| HIGH | 2 | 高，建议成功 |
| NORMAL | 3 | 普通，可选 |

## 4. 使用方式

### 4.1 注册新缓存

```python
# cache/my_cache.py
def warmup_my_cache() -> dict:
    """预热函数，返回结果字典"""
    # 预热逻辑
    return {'success': True, 'count': 100}

def init_cache():
    """注册到管理器"""
    from .manager import cache_manager, CacheConfig, WarmupMode, CachePriority
    
    cache_manager.register(CacheConfig(
        name="my_cache",
        warmup_func=warmup_my_cache,
        mode=WarmupMode.ASYNC,
        priority=CachePriority.NORMAL,
        timeout=60,
        retry=2
    ))
```

### 4.2 在 __init__.py 中注册

```python
# cache/__init__.py
def init_all_caches():
    from . import red_list
    red_list.init_cache()
    
    from . import stock_bond_mapping
    stock_bond_mapping.init_cache()
    
    from . import my_cache
    my_cache.init_cache()
```

### 4.3 在 app.py 中调用

```python
# app.py
def create_app():
    # ...
    from gs2026.dashboard2.cache import init_all_caches, cache_manager
    
    init_all_caches()
    results = cache_manager.warmup_all(sync_names=['red_list'])
    # ...
```
