# 数据采集时间机制设计文档

## 概述

本文档描述股票数据采集系统的时间控制机制，包括交易时间判断、集合竞价处理、定时采集逻辑等。

---

## 交易时间定义

### 标准交易时段

| 时段 | 时间范围 | 说明 |
|------|----------|------|
| 早盘集合竞价 | 09:25 - 09:30 | 开盘前集合竞价 |
| 上午交易 | 09:30 - 11:30 | 连续竞价 |
| 午休 | 11:30 - 13:00 | 非交易时间 |
| 下午交易 | 13:00 - 14:57 | 连续竞价 |
| 尾市集合竞价 | 14:57 - 15:00 | 收盘集合竞价 |

### 代码实现

```python
# 集合竞价时间段配置
AUCTION_PERIODS = [
    (dt_time(9, 25), dt_time(9, 30)),   # 早盘集合竞价
    (dt_time(14, 57), dt_time(15, 0)),  # 尾市集合竞价
]

# 交易时间判断
def is_trading_time(dt):
    if dt.weekday() >= 5:  # 周六、周日
        return False
    t = dt.time()
    # 上午交易时段（包含早盘集合竞价）
    if dt_time(9, 25) <= t <= dt_time(11, 30):
        return True
    # 下午交易时段（包含尾市集合竞价）
    if dt_time(13, 0) <= t <= dt_time(15, 0):
        return True
    return False
```

## 主循环设计

### 定时机制

采用 **3秒整数倍对齐** 的定时机制：

```python
INTERVAL = 3  # 轮询间隔（秒）

while True:
    now = time.time()
    # 计算下一个整数倍时刻
    next_time = ((now + interval) // interval) * interval
    sleep_seconds = next_time - now
    if sleep_seconds > 0:
        time.sleep(sleep_seconds)
    
    target_dt = datetime.fromtimestamp(next_time)
    process_func(target_dt)
```

**采集时间点：** 09:25:00, 09:25:03, 09:25:06, ..., 15:00:00

## 集合竞价处理

### 设计原则

集合竞价时段（09:25-09:30 和 14:57-15:00）数据变化频繁，但只需采集一次。

### 实现机制

```python
# 记录是否已在当前集合竞价时段获取过数据
_auction_data_fetched = {
    'morning': False,   # 9:25-9:30
    'afternoon': False, # 14:57-15:00
}

# 每日开盘前重置
if last_date != current_date:
    reset_auction_flags()
```

### 处理逻辑

```
检查是否在集合竞价时段？
    ├── 是
    │   ├── 是否已采集过？
    │   │   ├── 是 → continue（跳过）
    │   │   └── 否 → 采集数据，设置标志
    │   └── 【特殊】是否是15:00:00？
    │           └── 是 → 必须采集（收盘时刻）
    └── 否
            ↓
        正常交易时间处理
```

## 15:00:00 特殊处理

### 问题背景

尾市集合竞价时段（14:57-15:00）只采集一次数据，导致 15:00:00 被跳过。

### 修复方案

```python
if is_auction:
    # 【修复】尾市集合竞价的15:00:00必须采集
    if period_name == 'afternoon' and target_dt.time() == dt_time(15, 0):
        logger.info(f"[集合竞价] {target_dt.strftime('%H:%M:%S')} 尾市收盘，必须采集")
    elif _auction_data_fetched[period_name]:
        continue  # 已获取过，跳过
    else:
        _auction_data_fetched[period_name] = True
```

### 设计原则

- **15:00:00 是收盘时刻**，必须采集
- 即使已在集合竞价时段采集过数据，15:00:00 仍需单独采集
