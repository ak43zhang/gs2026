## 一、需求背景

在智能选股页面新增"涨停行概选股"子任务Tab。
核心流程：获取实时/历史涨停板数据 → 通过股票代码从宽表获取完整行业和概念 → 以标签面板形式展示 → 用户勾选后执行交叉选股。

---

## 二、数据源策略

### 2.1 双数据源

| 场景 | 数据源 | 表名 | 筛选条件 |
|------|--------|------|----------|
| 当天（默认） | 实时监控表 | `monitor_gp_sssj_{date}` | `is_zt = 1` (DISTINCT stock_code) |
| 历史日期 | 涨停分析表 | `analysis_ztb_detail_{year}` | `trade_date = '{date}'` |

### 2.2 数据流

```
  当天：monitor_gp_sssj_20260422 (is_zt=1)
         ↓ 提取去重stock_code (78只)
  历史：analysis_ztb_detail_2026 (trade_date)
         ↓ 提取去重stock_code (65-90只)
         ↓
  宽表 cache_stock_industry_concept_bond
         ↓ 通过stock_code获取完整行业+概念
         ↓
  统计去重 → 行业列表(带频次) + 概念列表(带频次)
         ↓
  前端展示标签面板（默认全部勾选）
         ↓ 用户确认后
  调用现有 query_cross_stocks() 执行交叉选股
```

### 2.3 行业/概念来源

**关键设计**：不使用涨停表自身的 `sectors`/`concepts` 字段（数据可能不完整），
而是通过涨停股票代码查询**宽表** `cache_stock_industry_concept_bond`，获取每只股票的**全部**行业和概念。

示例：
- 金证股份(600446) is_zt=1
- 宽表查到：行业=`IT服务`，概念=`AI应用`, `DeepSeek概念`, `华为概念` 等18个
- 全部纳入标签统计

## 三、API设计

### 3.1 新增API: `GET /api/stock-picker/ztb-tags`

**请求参数**:

| 参数 | 类型 | 必选 | 说明 |
|------|------|------|------|
| date | string | 否 | 日期(YYYYMMDD)，默认当天 |

**返回格式**:
```json
{
  "code": 0,
  "data": {
    "date": "20260422",
    "total_zt": 78,
    "industries": [
      {"name": "半导体", "type": "industry", "count": 8},
      {"name": "化学制品", "type": "industry", "count": 6}
    ],
    "concepts": [
      {"name": "AI应用", "type": "concept", "count": 12},
      {"name": "DeepSeek概念", "type": "concept", "count": 8}
    ]
  }
}
```

**字段说明**:
- `count`: 涨停股票中包含该行业/概念的股票数量
- `type`: 标签类型，用于传给 `query_cross_stocks()`
- 行业按count降序，概念按count降序

## 四、后端实现

### 4.1 服务层新增函数 (`stock_picker_service.py`)

```python
def get_ztb_tags(date: str = None) -> dict:
    """
    获取涨停板的行业和概念标签
    
    数据源策略：
    - 当天日期：从 monitor_gp_sssj_{date} 表查 is_zt=1
    - 历史日期：从 analysis_ztb_detail_{year} 表查 trade_date
    
    然后通过股票代码查宽表获取完整行业和概念
    """
    from datetime import datetime
    
    if not date:
        date = datetime.now().strftime('%Y%m%d')
    
    today = datetime.now().strftime('%Y%m%d')
    mysql_tool = mysql_util.get_mysql_tool()
    
    # 1. 获取涨停股票代码
    zt_codes = []
    
    if date == today:
        # 当天：从实时监控表查 is_zt=1
        try:
            with mysql_tool.engine.connect() as conn:
                sql = f"SELECT DISTINCT stock_code FROM monitor_gp_sssj_{date} WHERE is_zt = 1"
                rows = pd.read_sql(sql, conn).to_dict('records')
                zt_codes = [r['stock_code'] for r in rows]
        except Exception as e:
            logger.warning(f"实时监控表查询失败({date}): {e}")
    
    if not zt_codes:
        # 历史日期 或 当天实时表无数据：从涨停分析表查
        try:
            year = date[:4]
            table = f"analysis_ztb_detail_{year}"
            date_fmt = f"{date[:4]}-{date[4:6]}-{date[6:8]}"
            with mysql_tool.engine.connect() as conn:
                sql = f"SELECT DISTINCT stock_code FROM {table} WHERE trade_date = '{date_fmt}'"
                rows = pd.read_sql(sql, conn).to_dict('records')
                zt_codes = [r['stock_code'] for r in rows]
        except Exception as e:
            logger.error(f"涨停分析表查询失败({date}): {e}")
    
    if not zt_codes:
        return {'date': date, 'total_zt': 0, 'industries': [], 'concepts': []}
    
    # 2. 从宽表获取完整行业和概念
    # 确保内存缓存已加载
    if not _stock_cache:
        load_memory_cache()
    
    industry_counter = defaultdict(int)  # 行业名 → 出现次数
    concept_counter = defaultdict(int)   # 概念名 → 出现次数
    
    for code in zt_codes:
        stock_data = _stock_cache.get(code)
        if stock_data:
            for ind in stock_data['industries']:
                industry_counter[ind] += 1
            for con in stock_data['concepts']:
                concept_counter[con] += 1
    
    # 3. 按频次降序排列
    industries = sorted(
        [{'name': name, 'type': 'industry', 'count': count} 
         for name, count in industry_counter.items()],
        key=lambda x: x['count'], reverse=True
    )
    
    concepts = sorted(
        [{'name': name, 'type': 'concept', 'count': count} 
         for name, count in concept_counter.items()],
        key=lambda x: x['count'], reverse=True
    )
    
    return {
        'date': date,
        'total_zt': len(zt_codes),
        'industries': industries,
        'concepts': concepts
    }
```

### 4.2 路由层新增 (`stock_picker.py`)

```python
@stock_picker_bp.route('/api/stock-picker/ztb-tags')
def ztb_tags():
    """获取涨停板行业概念标签"""
    date = request.args.get('date')
    try:
        result = stock_picker_service.get_ztb_tags(date)
        return jsonify({'code': 0, 'data': result})
    except Exception as e:
        logger.error(f"获取涨停标签失败: {e}")
        return jsonify({'code': 1, 'message': str(e)})
```

### 5.3 交互流程

1. 切换到"涨停行概选股"Tab
2. 默认加载当天数据，调用 `/api/stock-picker/ztb-tags`
3. 渲染行业标签面板 + 概念标签面板，**默认全部勾选**
4. 日期切换 → 重新调用 `/api/stock-picker/ztb-tags?date=YYYYMMDD`
5. 全选/取消 按钮控制整体勾选
6. 点击"查询" → 将已勾选标签组装为 `query_cross_stocks()` 参数调用 `/api/stock-picker/query`
7. 结果展示**完全复用**交叉行概选股的渲染逻辑

### 5.4 标签传递格式

涨停标签面板勾选后，传给 `/api/stock-picker/query` 的格式需与现有一致。

**问题**：现有接口使用 `industry:881121` 格式（code），但涨停标签只有 name 没有 code。

**解决方案**：新增一个查询入口 `/api/stock-picker/query-by-names`，直接用名称匹配。
或者在 `get_ztb_tags()` 返回中包含 code，从宽表反查。

**选择**：在 `get_ztb_tags()` 返回中包含 code。修改逻辑：
- 行业：从搜索器 `_pinyin_searcher` 中查找 name → code 映射
- 概念：同上
- 若找不到 code，仍用 name 作为 code

这样前端可以直接用现有 `/api/stock-picker/query?tags=industry:881121,concept:886055,...` 接口。
