# 行业上攻排行联动筛选功能设计

**文档版本**: v1.0  
**创建日期**: 2026-04-17  
**功能模块**: 数据监控页面 - 上攻排行  
**相关文件**: `src/gs2026/dashboard2/templates/monitor.html`

---

## 一、需求描述

在数据监控页面的行业上攻排行中，点击某个行业后，股票上攻排行和债券上攻排行自动筛选出该行业对应的股票和债券数据。

---

## 二、交互设计

### 2.1 点击行为

| 操作 | 效果 |
|------|------|
| 点击某行业 | 股票排行和债券排行只显示该行业的数据，被选行业高亮 |
| 再次点击同一行业 | 取消筛选，恢复显示全部数据 |
| 点击另一个行业 | 切换到新行业的筛选 |

### 2.2 视觉反馈

- **行业排行**：被选中的行业行添加高亮背景色 `#e8f0fe`
- **股票/债券排行标题**：筛选激活时显示筛选标签 `行业：XX ✕`
- **取消筛选**：点击标签上的 `✕` 或再次点击已选行业

### 2.3 交互流程

```
用户点击行业"AI算力"
  ├── 行业排行: "AI算力"行高亮
  ├── 股票排行标题: 显示"行业：AI算力 ✕"
  ├── 股票排行内容: 只显示 industry_name === "AI算力" 的股票
  ├── 债券排行标题: 显示"行业：AI算力 ✕"
  └── 债券排行内容: 只显示 industry_name === "AI算力" 的债券

用户再次点击"AI算力"（或点击 ✕）
  ├── 行业排行: 取消高亮
  ├── 股票/债券排行标题: 隐藏筛选标签
  └── 股票/债券排行内容: 恢复显示全部数据
```

## 三、技术设计

### 3.1 实现方式

**纯前端实现**，不修改后端API。

### 3.2 新增状态变量

```javascript
let _selectedIndustry = null;  // 当前选中的行业名称，null表示未筛选
```

### 3.3 新增函数

#### `selectIndustry(name)`
行业点击事件处理函数：
```javascript
function selectIndustry(name) {
    // 切换选中状态（点击同一行业取消选中）
    _selectedIndustry = (_selectedIndustry === name) ? null : name;
    
    // 更新行业排行高亮
    updateIndustryHighlight();
    
    // 更新筛选标签显示
    updateIndustryFilterTag();
    
    // 重新筛选并渲染股票排行
    rerenderStockRanking();
    
    // 重新筛选并渲染债券排行
    rerenderBondRanking();
}
```

#### `updateIndustryHighlight()`
更新行业排行中选中行的高亮样式。

#### `updateIndustryFilterTag()`
更新股票/债券排行标题旁的筛选标签显示/隐藏。

#### `rerenderStockRanking()` / `rerenderBondRanking()`
从 `_rankRawData` 原始数据中过滤并重新渲染排行。

### 3.4 筛选逻辑

```javascript
function filterByIndustry(data) {
    if (!_selectedIndustry) return data;
    return data.filter(item => item.industry_name === _selectedIndustry);
}
```

筛选流程：
```
原始数据 (_rankRawData)
  → 行业筛选 (filterByIndustry)
  → 前N行业筛选 (filterBondByTopNSectors，仅债券)
  → 排序 (sortRankData)
  → 渲染 (renderRanking)
```

### 3.5 HTML 修改

#### 行业排行行添加点击事件
```html
<tr onclick="selectIndustry('${name}')" style="cursor:pointer">
```

#### 股票/债券排行标题添加筛选标签
```html
<span id="stock-industry-tag" style="display:none;margin-left:8px;font-size:12px;

</span>
```

## 四、兼容性处理

### 4.1 与现有功能的兼容

| 现有功能 | 兼容方式 |
|---------|---------|
| 涨跌幅排序 | 筛选后仍可排序，互不影响 |
| 次数排序 | 筛选后仍可排序，互不影响 |
| 行业排序 | 筛选后仍可排序，互不影响 |
| 前N行业筛选（债券） | 行业筛选和前N筛选可叠加 |
| 定时数据刷新 | 刷新后保持当前筛选状态 |

### 4.2 数据刷新保持筛选

定时轮询获取新数据后，在渲染前自动应用当前的行业筛选条件：
```javascript
// loadStockRanking / loadBondRanking 中
_rankRawData['stock-ranking'] = newData;
const filtered = filterByIndustry(newData);
renderRanking('stock-ranking', filtered);
```
