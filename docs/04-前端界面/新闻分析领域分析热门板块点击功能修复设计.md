## 一、问题描述

### 问题 1：点击热门板块后数据未按评分倒序排序

**现象**: 用户点击热点板块列表中的某个板块后，展示的新闻/领域数据没有按照评分从高到低排序。

**根本原因**:
- `filterBySector()` 函数调用 API 时没有传递 `sort` 参数
- 后端默认按 `time`（时间）排序
- 用户期望看到该板块下评分最高的事件

### 问题 2：点击板块后自动刷新未保持筛选条件

**现象**: 点击板块筛选后，30秒自动刷新时会重置为默认列表，丢失板块筛选条件。

**根本原因**:
- `setInterval` 只调用 `loadNews()`，使用全局默认变量
- 没有记录当前正在查看的板块筛选状态
- 刷新时未携带板块参数

## 二、修复方案

### 2.1 新闻分析修复 (`news.html`)

#### 新增状态变量
```javascript
let currentSector = '';  // 当前筛选的板块名称
```

#### 修改 `filterBySector` 函数
```javascript
function filterBySector(sector) {
    currentPage = 1;
    currentSector = sector;  // 记录当前板块
    currentSort = 'score';   // 按评分倒序排序
    
    let url = `/api/news/list?page=1&page_size=${PAGE_SIZE}&sector=${encodeURIComponent(sector)}&sort=${currentSort}`;
    if (selectedDate) {
        url += `&date=${selectedDate}`;
    }
    // ... 其余代码不变
}
```

#### 修改 `resetFilter` 函数
```javascript
function resetFilter() {
    currentType = ''; 
    currentSize = ''; 
    currentSort = 'time';   // 重置为时间排序
    currentSector = '';      // 清除板块筛选
    currentPage = 1;
    // ... 其余代码不变
}
```

#### 修改 `loadNews` 函数
```javascript
function loadNews(append = false) {
    let url = `/api/news/list?page=${currentPage}&page_size=${PAGE_SIZE}&sort=${currentSort}`;
    
    // 添加日期参数
    if (selectedDate) {
        url += `&date=${selectedDate}`;
    }
    
    // 添加板块筛选参数
    if (currentSector) {
        url += `&sector=${encodeURIComponent(currentSector)}`;
    }
    
    // ... 其余参数不变
}
```

#### 修改自动刷新逻辑
```javascript
// 30秒自动刷新 - 保持当前筛选条件
setInterval(() => { 
    loadStats(); 
    loadHotSectors();
    // 仅在非板块筛选模式下自动刷新列表
    if (!currentSector) {
        loadNews(); 
    }
}, 30000);
```
