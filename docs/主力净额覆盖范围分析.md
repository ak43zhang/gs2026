### 代码逻辑分析

**位置**: `calculate_main_force_and_cumulative()` 函数

**问题1：inner join导致股票丢失**
```python
merged = pd.merge(
    df_now[['stock_code', 'short_name', 'price', 'volume', 'amount', 'change_pct', 'is_zt']],
    df_prev_main[['stock_code', 'volume', 'amount', 'change_pct']],
    on='stock_code',
    suffixes=('_now', '_prev'),
    how='inner'  # ← 问题：只保留两边都有的股票
)
```

**效果**: 
- `df_now` 有5000只股票
- `df_prev_main` 可能有4800只（部分股票缺失）
- `inner` join后只剩4800只
- **200只股票丢失**

---

**问题2：门槛过滤进一步减少股票**
```python
mask = (merged['delta_amount'] >= MAIN_FORCE_CONFIG['min_amount']) & \
       (merged['delta_volume'] >= MAIN_FORCE_CONFIG['min_volume'])
valid_data = merged[mask].copy()
```

**配置**:
```python
MAIN_FORCE_CONFIG = {
    'min_amount': 300000,   # 30万
    'min_volume': 20000,    # 200手
}
```

**效果**:
- 只有成交额变化≥30万且成交量≥200手的股票才被计算
- 大部分股票的成交额变化只有几万到几十万
- **90%以上股票被过滤**

---

**问题3：empty判断直接返回**
```python
if valid_data.empty:
    return df_now  # ← 直接返回，所有股票都是初始值0

# 只有满足门槛的股票才会执行到这里
calculate_main_force_behavior(...)
calculate_participation_ratio(...)
```

**效果**:
- 被过滤的股票：`main_net_amount=0`, `main_behavior='无主力'`, `main_confidence=0`
- 只有满足门槛的股票才有计算值

### 当前实际效果

假设 `df_now` 有5000只股票：

| 阶段 | 股票数量 | 说明 |
|------|----------|------|
| df_now | 5000 | 当前时刻所有股票 |
| inner join后 | ~4800 | 丢失200只（df_prev_main中缺失） |
| 门槛过滤后 | ~200 | 只有成交额大的股票 |
| 最终计算 | ~200 | 只有这200只有主力净额值 |
| 被过滤股票 | ~4600 | 主力净额=0，行为='无主力' |

**结论**: 当前只有约200只股票（4%）有主力净额计算，96%的股票显示为0/'无主力'

### 方案D：分层计算（精细控制）

**思路**: 根据成交额大小分层处理

```python
# 分层门槛
TIER_1 = 1000000   # 100万 - 正常计算
TIER_2 = 100000    # 10万 - 简化计算
TIER_3 = 0         # 小于10万 - 主力净额=0

# 分层处理
for tier, threshold in [(TIER_1, 1000000), (TIER_2, 100000), (TIER_3, 0)]:
    mask = (merged['delta_amount'] >= threshold) & (merged['delta_amount'] < tier)
    tier_data = merged[mask].copy()
    
    if threshold == TIER_1:
        # 正常计算
        tier_data['main_net_amount'] = ...
    elif threshold == TIER_2:
        # 简化计算（降低参与系数）
        tier_data['main_net_amount'] = tier_data['delta_amount'] * 0.1
    else:
        # 主力净额=0
        tier_data['main_net_amount'] = 0
    
    # 合并结果
```

**优点**: 精细控制，小成交额股票也有值
**缺点**: 复杂度高，需要分层逻辑
