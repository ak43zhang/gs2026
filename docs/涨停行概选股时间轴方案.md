# 涨停行概选股时间轴方案设计

## 一、需求理解

### 当前现状
- 涨停行概选股页面有日期选择器
- 选择日期后显示该日涨停的股票
- 缺少时间维度筛选

### 用户期望
- **在日期选择器下方增加时间轴**
- **时间轴逻辑与数据监控页面一致**
- **选择时间点+刷新后，显示该时间点的涨停股票**
- **后续逻辑保持一致**

### 关键问题
涨停是**时间段概念**（一旦涨停，后续时间都是涨停状态），不是**时间点概念**。需要明确：
1. "10:00的涨停股票"是指什么？
2. 是指10:00**首次**涨停的股票？
3. 还是指10:00**已经**涨停的股票（包含更早涨停的）？

---

## 二、方案设计

### 方案A：首次涨停时间筛选（推荐）

**逻辑**: 显示在选定时间点**首次涨停**的股票

**示例**:
| 股票 | 首次涨停时间 | 09:30筛选 | 10:00筛选 | 14:00筛选 |
|------|------------|----------|----------|----------|
| A | 09:30 | ✅ 显示 | ❌ 不显示 | ❌ 不显示 |
| B | 10:00 | ❌ 不显示 | ✅ 显示 | ❌ 不显示 |
| C | 14:00 | ❌ 不显示 | ❌ 不显示 | ✅ 显示 |

**适用场景**: 查看"某时刻新涨停的股票"

**优点**:
- 清晰明确，每只股票只出现一次
- 便于分析涨停时间分布
- 与"涨停时间线"概念一致

**缺点**:
- 不显示已涨停但未新涨停的股票

---

### 方案B：累计涨停状态筛选

**逻辑**: 显示在选定时间点**已经涨停**的所有股票

**示例**:
| 股票 | 首次涨停时间 | 09:30筛选 | 10:00筛选 | 14:00筛选 |
|------|------------|----------|----------|----------|
| A | 09:30 | ✅ 显示 | ✅ 显示 | ✅ 显示 |
| B | 10:00 | ❌ 不显示 | ✅ 显示 | ✅ 显示 |
| C | 14:00 | ❌ 不显示 | ❌ 不显示 | ✅ 显示 |

**适用场景**: 查看"某时刻市场有多少涨停股"

**优点**:
- 反映市场热度
- 可以看到涨停板的累积过程

**缺点**:
- 同一只股票在不同时刻都会显示
- 与"首次涨停"概念不同

---

### 方案C：时间区间筛选（折中）

**逻辑**: 显示在**选定时间区间内**首次涨停的股票

**示例**: 选择09:30-10:00，显示该区间首次涨停的股票

**适用场景**: 分析特定时间段的新涨停股票

---

## 三、推荐方案：方案A（首次涨停时间）

### 理由
1. **业务语义清晰**: "10:00涨停"通常指"10:00新涨停"
2. **数据一致性**: 每只股票只出现一次，不重复
3. **分析价值高**: 便于研究涨停时间分布规律
4. **实现简单**: 基于现有的`first_zt_time`字段

### 数据模型

假设已有数据：
```sql
-- 涨停记录表（已存在）
CREATE TABLE zt_records (
    stock_code VARCHAR(10),
    stock_name VARCHAR(20),
    first_zt_time TIME,  -- 首次涨停时间
    zt_count INT,        -- 涨停次数
    ...
);
```

### 筛选逻辑

```python
def get_ztb_list_by_time(date, time_str=None):
    """
    获取涨停列表（支持时间筛选）
    
    Args:
        date: 日期
        time_str: 时间字符串，如 "10:00"，None表示全天
    
    Returns:
        涨停股票列表
    """
    base_query = f"""
        SELECT * FROM zt_records_{date}
        WHERE 1=1
    """
    
    if time_str:
        # 方案A：首次涨停时间在time_str之前（或等于）
        # 例如 time_str="10:00"，显示first_zt_time <= "10:00"的股票
        # 但只显示最近一次刷新时间点新涨停的？
        # 需要更精确的定义...
        
        # 精确逻辑：显示在(time_str - 3min, time_str]区间内首次涨停的股票
        # 因为数据采集周期是3分钟
        base_query += f"""
            AND first_zt_time > TIME('{time_str}') - INTERVAL 3 MINUTE
            AND first_zt_time <= TIME('{time_str}')
        """
    
    return execute_query(base_query)
```

### 更精确的逻辑

考虑到数据采集周期是3分钟，时间轴应该显示：
- **该时间点新采集到的涨停股票**
- 即：`first_zt_time`在`(prev_time, current_time]`区间内的股票

```python
def get_ztb_list_by_time(date, current_time, prev_time=None):
    """
    获取指定时间点的涨停股票（新涨停）
    
    Args:
        date: 日期
        current_time: 当前时间，如 "10:00:00"
        prev_time: 上一时间，如 "09:57:00"，None表示从开盘开始
    
    Returns:
        在该时间窗口内首次涨停的股票
    """
    if prev_time is None:
        # 从开盘到current_time
        condition = f"first_zt_time <= '{current_time}'"
    else:
        # 在(prev_time, current_time]区间内首次涨停
        condition = f"first_zt_time > '{prev_time}' AND first_zt_time <= '{current_time}'"
    
    query = f"""
        SELECT * FROM zt_records_{date}
        WHERE {condition}
        ORDER BY first_zt_time
    """
    return execute_query(query)
```

### 服务层修改

```python
# ztb_analysis_service.py

def get_ztb_list(date, time_str=None, ...):
    """
    获取涨停列表
    
    Args:
        date: 日期
        time_str: 时间字符串，如 "10:00:00"
                None表示全天，不筛选时间
    """
    table_name = f"zt_records_{date.replace('-', '')}"
    
    base_sql = f"""
        SELECT * FROM {table_name}
        WHERE 1=1
    """
    
    if time_str:
        # 方案A：显示在该时间点新涨停的股票
        # 即first_zt_time在(prev_time, time_str]区间内
        prev_time = calculate_prev_time(time_str)  # 根据3分钟周期计算
        base_sql += f"""
            AND first_zt_time > '{prev_time}'
            AND first_zt_time <= '{time_str}'
        """
    
    # ... 其他筛选条件
    
    return execute_query(base_sql)

def calculate_prev_time(time_str, interval_minutes=3):
    """
    计算上一个时间点
    
    例如：
    - "10:00:00" -> "09:57:00" (3分钟间隔)
    - "09:30:00" -> "09:30:00" (开盘时间)
    """
    from datetime import datetime, timedelta
    
    t = datetime.strptime(time_str, '%H:%M:%S')
    
    # 如果早于09:30，返回09:30
    if t.hour < 9 or (t.hour == 9 and t.minute < 30):
        return '09:30:00'
    
    # 减去间隔
    prev = t - timedelta(minutes=interval_minutes)
    
    # 处理午休时间
    if prev.hour == 11 and prev.minute >= 30:
        # 11:30之后跳到13:00
        return '11:30:00'
    if prev.hour == 13 and prev.minute < 30:
        # 13:00-13:30之间，跳到11:30
        return '11:30:00'
    
    return prev.strftime('%H:%M:%S')
```
