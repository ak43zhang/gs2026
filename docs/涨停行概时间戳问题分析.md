# 涨停行概时间戳问题分析

## 问题描述
Redis中没有时间戳数据：`monitor_gp_apqd_20260429:timestamps`

## 原因分析

### 1. 时间戳存储逻辑

从代码分析，时间戳是通过`save_dataframe_to_redis`函数存储的：

```python
def save_dataframe_to_redis(df, table_name, time_full, expire_seconds, use_compression=False):
    # ...
    pipe = client.pipeline()
    pipe.lpush(f"{table_name}:timestamps", time_full)  # 添加时间戳到列表
    pipe.ltrim(f"{table_name}:timestamps", 0, MAX_SAVE_TIMESTAMPS - 1)  # 限制列表长度
    pipe.expire(f"{table_name}:timestamps", expire_seconds)  # 设置过期时间
    pipe.execute()
```

### 2. 问题定位

**可能原因1：Redis数据已过期**
- 过期时间是 `EXPIRE_SECONDS = 64800` 秒（约18小时）
- 如果数据采集已经停止超过18小时，Redis数据会过期

**可能原因2：异步存储失败**
- `save_dataframe_async`使用后台线程池存储
- 如果线程池任务失败，数据不会写入Redis

**可能原因3：Redis从未写入**
- 如果监控服务没有正常运行，Redis中不会有数据

### 3. 解决方案

当Redis中没有时间戳数据时，需要从MySQL获取。

**当前代码问题：**
```python
def get_ztb_timestamps(date: str) -> List[str]:
    # 1. 尝试从Redis获取
    # ...
    
    # 2. 如果Redis没有，直接返回空列表（没有MySQL回退）
    return []
```

**需要添加MySQL回退：**
```python
def get_ztb_timestamps(date: str) -> List[str]:
    # 1. 尝试从Redis获取
    # ...
    
    # 2. 从MySQL获取（作为回退）
    # 查询 monitor_gp_sssj 表的时间列
```

## 实施步骤

### 方案A：添加MySQL回退（推荐）

当Redis中没有时间戳时，从`monitor_gp_sssj`表的`time`列获取：

```python
def get_ztb_timestamps(date: str) -> List[str]:
    try:
        date_str = date.replace('-', '')
        
        # 1. 尝试从Redis获取
        redis_key = f"monitor_gp_apqd_{date_str}:timestamps"
        # ...
        
        # 2. 从MySQL获取（回退）
        table_name = f"monitor_gp_sssj_{date_str}"
        sql = f"SELECT DISTINCT time FROM {table_name} ORDER BY time"
        df = pd.read_sql(sql, engine)
        if not df.empty:
            timestamps = df['time'].tolist()
            return timestamps
        
        return []
    except Exception as e:
        logger.error(f"获取时间戳失败: {e}")
        return []
```

### 方案B：使用monitor_gp_sssj的时间戳

直接使用`monitor_gp_sssj`表的时间戳，不依赖`monitor_gp_apqd`：

```python
def get_ztb_timestamps(date: str) -> List[str]:
    try:
        date_str = date.replace('-', '')
        redis_key = f"monitor_gp_sssj_{date_str}:timestamps"  # 使用sssj的时间戳
        # ...
    except Exception as e:
        logger.error(f"获取时间戳失败: {e}")
        return []
```

## 建议

**推荐方案A**，因为：
1. `monitor_gp_sssj`表一定有数据（实时存储）
2. `time`列一定存在
3. 可以获取所有时间点
