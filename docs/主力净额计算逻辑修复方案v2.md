# 主力净额计算逻辑修复方案 v2

## 核心理解

### 业务现状
- **上攻排行周期**: 15秒（固定业务需求）
- **主力净额计算**: 应该独立走"最新时间戳"逻辑
- **关键区别**: 主力净额计算不受15秒周期限制，找"上一个有数据的时间点"

### 当前问题
```
上攻排行: 每15秒计算一次（09:30:00, 09:30:15, 09:30:30...）
主力净额: 也按15秒计算（导致时间断层时无数据）

问题场景:
09:30:00 - 有数据 ✓
09:30:03 - 有数据 ✓  
09:30:15 - 无数据（系统延迟）
09:30:30 - 有数据，但找15秒前(09:30:15)无数据 → 主力净额=0 ✗

期望:
09:30:30 - 找上一个有数据的时间点(09:30:03) → 正常计算 ✓
```

---

## 修复方案

### 设计原则
1. **上攻排行周期保持15秒不变**（业务需求）
2. **主力净额计算独立**：找"上一个有数据的时间点"
3. **不影响其他字段**：只修改主力净额相关逻辑

### 修改点1：新增函数 - 获取上一个有数据的时间点

**文件**: `monitor_stock.py`

```python
def get_prev_timestamp_with_data(table_name: str, current_time: str, redis_client=None) -> str:
    """
    获取上一个有数据的时间点（用于主力净额计算）
    
    与上攻排行的15秒周期解耦，独立找最新数据
    
    Args:
        table_name: 表名如 monitor_gp_sssj_20260429
        current_time: 当前时间 HH:MM:SS
        redis_client: Redis客户端
    
    Returns:
        上一个有数据的时间点，如 "09:30:03"，无则返回None
    """
    prev_time = None
    
    # 方法1: 从Redis时间戳列表获取
    if redis_client:
        try:
            ts_key = f"{table_name}:timestamps"
            all_ts = redis_client.lrange(ts_key, 0, -1)
            
            if all_ts:
                # 解码并排序
                timestamps = sorted([
                    t.decode('utf-8') if isinstance(t, bytes) else t
                    for t in all_ts
                ])
                
                # 找当前时间之前的最新时间
                for ts in reversed(timestamps):
                    if ts < current_time:
                        prev_time = ts
                        break
        except Exception as e:
            logger.warning(f"从Redis获取时间戳失败: {e}")
    
    # 方法2: 从MySQL查询（备用）
    if not prev_time:
        try:
            query = f"""
                SELECT MAX(time) as prev_time
                FROM {table_name}
                WHERE time < '{current_time}'
                LIMIT 1
            """
            result = pd.read_sql(query, con=engine)
            if not result.empty and result.iloc[0]['prev_time']:
                prev_time = result.iloc[0]['prev_time']
        except Exception as e:
            logger.warning(f"从MySQL获取上一时间失败: {e}")
    
    return prev_time


def load_dataframe_by_time(table_name: str, time_str: str) -> pd.DataFrame:
    """
    根据时间点加载DataFrame
    
    优先Redis，其次MySQL
    """
    # 1. 尝试Redis
    redis_key = f"{table_name}:{time_str}"
    df = redis_util.load_dataframe_by_key(redis_key, use_compression=False)
    
    if df is not None and not df.empty:
        return df
    
    # 2. 回退MySQL
    try:
        query = f"""
            SELECT * FROM {table_name}
            WHERE time = '{time_str}'
        """
        df = pd.read_sql(query, con=engine)
        return df
    except Exception as e:
        logger.error(f"从MySQL加载数据失败: {e}")
        return pd.DataFrame()
```

### 修改点3：修改累计主力净额计算

**文件**: `monitor_stock.py` 第158行

```python
def calculate_cumulative_main_net_v2(df: pd.DataFrame, 
                                     table_name: str, 
                                     current_time: str,
                                     redis_client=None) -> pd.DataFrame:
    """
    计算累计主力净额（改进版）
    
    使用"上一个有累计值的时间点"，而非固定上一时刻
    """
    # 初始化累计值为当前值
    df['cumulative_main_net'] = df['main_net_amount'].fillna(0)
    
    try:
        # 【修改】找上一个有累计值的时间点
        prev_time = get_prev_timestamp_with_data(table_name, current_time, redis_client)
        
        if prev_time:
            # 查询该时间点的累计值
            stock_codes = df['stock_code'].tolist()
            codes_str = ','.join([f"'{c}'" for c in stock_codes])
            
            query = f"""
                SELECT stock_code, cumulative_main_net
                FROM {table_name}
                WHERE time = '{prev_time}' 
                AND stock_code IN ({codes_str})
                AND cumulative_main_net != 0
            """
            
            prev_cumulative = pd.read_sql(query, con=engine)
            
            if not prev_cumulative.empty:
                # 类型统一
                df['stock_code'] = df['stock_code'].astype(str)
                prev_cumulative['stock_code'] = prev_cumulative['stock_code'].astype(str)
                
                # 合并
                df = df.merge(
                    prev_cumulative[['stock_code', 'cumulative_main_net']],
                    on='stock_code',
                    how='left',
                    suffixes=('', '_prev')
                )
                
                # 新的累计值 = 上一累计值 + 当前值
                df['cumulative_main_net_prev'] = df['cumulative_main_net_prev'].fillna(0)
                df['cumulative_main_net'] = df['cumulative_main_net_prev'] + df['main_net_amount'].fillna(0)
                df = df.drop(columns=['cumulative_main_net_prev'], errors='ignore')
                
                logger.info(f"[{current_time}] 累计值基于{prev_time}计算，共{len(prev_cumulative)}只股票")
        else:
            logger.info(f"[{current_time}] 无上一累计值，使用当前值")
            
    except Exception as e:
        logger.error(f"[{current_time}] 计算累计主力净额失败: {e}")
        df['cumulative_main_net'] = df['main_net_amount'].fillna(0)
    
    return df
```

## 关键区别

| 功能 | 时间逻辑 | 数据来源 |
|------|---------|---------|
| **上攻排行** | 固定15秒周期 | df_prev (15秒前) |
| **主力净额** | 最新时间戳 | df_prev_main (上一个有数据点) |
| **累计净额** | 最新时间戳 | 查询上一个有累计值的时间点 |
