# 数据类型统一优化方案

## 问题分析

当前问题：`calculate_main_force_and_cumulative`和`calculate_top30_v3`中出现`merge`时列类型不匹配（object vs int64）。

**根本原因：**
1. `normalize_stock_dataframe`在`deal_gp_works`入口统一清洗`df_now`
2. 但`df_prev_main`是从Redis加载的历史数据，**没有经过统一清洗**
3. 导致`df_now['stock_code']`是字符串，`df_prev_main['stock_code']`可能是int64

## 当前修复方式

在每个使用`merge`的函数中手动转换：
```python
df_now['stock_code'] = df_now['stock_code'].astype(str)
df_prev_main['stock_code'] = df_prev_main['stock_code'].astype(str)
```

**问题：** 重复代码，容易遗漏

## 优化方案

### 方案A：在Redis加载时统一清洗（推荐）

修改`load_dataframe_by_time`和`load_dataframe_by_key`函数，在加载后立即清洗：

```python
def load_dataframe_by_time(table: str, time_str: str, use_compression: bool = False) -> pd.DataFrame:
    """加载指定时间点的DataFrame"""
    df = _load_from_redis_or_mysql(table, time_str, use_compression)
    if df is not None and not df.empty:
        # 【新增】统一清洗
        df = normalize_stock_dataframe(df, required_cols=['stock_code'])
    return df

def load_dataframe_by_key(key: str, use_compression: bool = False) -> pd.DataFrame:
    """通过key加载DataFrame"""
    df = _load_from_redis(key, use_compression)
    if df is not None and not df.empty:
        # 【新增】统一清洗
        df = normalize_stock_dataframe(df, required_cols=['stock_code'])
    return df
```

**优点：**
- 一处修改，全局生效
- 所有从Redis/MySQL加载的数据都经过统一清洗

**缺点：**
- 需要修改Redis工具函数
- 可能影响其他使用这些函数的地方

### 方案B：在calculate函数入口统一清洗

在`calculate_main_force_and_cumulative`和`calculate_top30_v3`入口统一清洗输入DataFrame：

```python

        df_prev = normalize_stock_dataframe(df_prev, required_cols=['stock_code'])
    
    # 原有逻辑...
```

**优点：**
- 只影响特定函数，风险可控
- 与P2-B优化一致（统一清洗入口）

**缺点：**
- 每个calculate函数都需要添加
- 可能重复清洗（如果输入已经清洗过）

### 方案C：在deal_gp_works中统一清洗所有df

在`deal_gp_works`中，对所有加载的DataFrame进行清洗：

```python
def deal_gp_works(loop_start):
    # ...
    
    # 【P2-B】统一清洗df_now
    if USE_UNIFIED_CLEAN:
        df_now = normalize_stock_dataframe(df_now, required_cols=['stock_code', 'price'])
    
    # 【新增】统一清洗df_prev
    if df_prev is not None and USE_UNIFIED_CLEAN:
        df_prev = normalize_stock_dataframe(df_prev, required_cols=['stock_code'])
    
    # 【新增】统一清洗df_prev_main
    if df_prev_main is not None and USE_UNIFIED_CLEAN:
        df_prev_main = normalize_stock_dataframe(df_prev_main, required_cols=['stock_code'])
    
    # 后续计算...
```

**优点：**
- 一处修改，影响所有下游函数
- 与现有P2-B优化风格一致

**缺点：**
- 需要确保所有df都经过清洗
- 可能重复清洗

## 推荐方案：方案C

理由：
1. 与P2-B优化风格一致（统一清洗入口）
2. 一处修改，全局生效
3. 风险可控（只在deal_gp_works中修改）
