# 涨停行概时间轴问题分析V2

## 问题确认

根据代码分析，`df_now`（即`monitor_gp_sssj`表）包含以下列：
- `stock_code` - 股票代码
- `short_name` - 股票简称
- `price` - 价格
- `change` - 涨跌额
- `change_pct` - 涨跌幅
- `volume` - 成交量
- `amount` - 成交额
- `time` - 时间（HH:MM:SS）
- `is_zt` - 是否涨停（0/1）
- `ever_zt` - 是否曾经涨停
- `is_auction` - 是否集合竞价
- `main_net_amount` - 主力净额
- `main_behavior` - 主力行为
- `main_confidence` - 主力置信度
- `cumulative_main_net` - 累计主力净额

## 问题定位

### 问题1: MySQL查询列名错误

**当前代码：**
```python
sql = f"SELECT * FROM {table_name} WHERE time_str = '{time}' AND is_zt = 1"
```

**实际列名：**
- 时间列是 `time`，不是 `time_str`
- 涨停列是 `is_zt`，正确

### 问题2: Redis key格式

**当前代码：**
```python
redis_key = f"monitor_gp_sssj_{date_str}:{time}"
```

**实际格式：**
- 正确，与`save_dataframe_to_redis`函数中的格式一致

## 解决方案

### 修复MySQL查询列名

将`time_str`改为`time`：

```python
# get_ztb_snapshot函数
sql = f"SELECT * FROM {table_name} WHERE time = '{time}' AND is_zt = 1"

# filter_ztb_snapshot函数
sql = f"SELECT * FROM {table_name} WHERE time = '{time}' AND is_zt = 1"
```
