# 债券排行无涨跌幅问题排查记录

## 问题描述

### 现象1：时间轴点击无数据
- 点击时间轴特定时间（如 10:25:27）时，债券上攻排行没有涨跌幅和行业数据
- 实时刷新时数据正常

### 现象2：实时场景无涨跌幅
- 实时场景下，债券上攻排行没有涨跌幅
- 股票上攻排行的涨跌幅正常

---

## 排查过程

### 第一阶段：定位问题接口

**检查前端请求：**
```javascript
// 时间轴点击时调用
loadRankingAtTime('bond', timeStr)
// 请求 URL: /api/monitor/attack-ranking/bond/at-time?date=20260401&time=10:25:27&limit=15
```

**检查后端接口：**
```python
@monitor_bp.route('/attack-ranking/<asset_type>/at-time', methods=['GET'])
def get_ranking_at_time(asset_type):
    # 调用 data_service.get_ranking_at_time()
    # 没有调用 _enrich_bond_data()
```

**结论：** `at-time` 接口没有为债券数据添加涨跌幅和行业信息。

---

### 第二阶段：分析数据流

**实时场景数据流：**
```
get_bond_ranking()
    ↓
data_service.get_bond_ranking()  # 获取基础数据
    ↓
_enrich_bond_data()              # 【有】添加涨跌幅和行业
    ↓
返回完整数据
```

**时间轴场景数据流：**
```
get_ranking_at_time()
    ↓
data_service.get_ranking_at_time()  # 获取基础数据
    ↓
【缺少】_enrich_bond_data()         # 没有添加涨跌幅和行业
    ↓
返回不完整数据
```

**结论：** 需要为 `at-time` 接口添加债券数据增强逻辑。

---

### 第三阶段：修复 at-time 接口

**修复代码：**
```python
@monitor_bp.route('/attack-ranking/<asset_type>/at-time', methods=['GET'])
def get_ranking_at_time(asset_type):
    # ... 获取基础数据 ...
    data = data_service.get_ranking_at_time(...)
    
    # 【修复】为债券数据添加涨跌幅和行业信息
    if asset_type == 'bond' and data and time_str:
        actual_date = date or datetime.now().strftime('%Y%m%d')
        data = _enrich_bond_data(data, actual_date, time_str)
    
    return jsonify({'success': True, 'data': data})
```

**提交：** `0310771` fix: 修复at-time接口债券数据缺少涨跌幅和行业的问题

---

### 第四阶段：排查实时场景问题

**测试实时场景：**
```bash
$ python test_realtime.py

=== 测试债券排行（实时场景）===
MySQL降级查询债券涨跌幅失败: type object 'Config' has no attribute 'MYSQL_URI'
第一条: code=113618, change_pct=-, industry=-

=== 测试股票排行（实时场景）===
第一条: code=002263, change_pct=5.58
```

**关键错误：**
```
MySQL降级查询债券涨跌幅失败: type object 'Config' has no attribute 'MYSQL_URI'
```

**分析：**
- 债券涨跌幅从 Redis 获取失败时，会降级查询 MySQL
- `_get_bond_change_pct_from_mysql` 引用 `Config.MYSQL_URI`
- `dashboard2/config.py` 中没有 `MYSQL_URI` 配置

---

### 第五阶段：修复 Config 配置

**问题代码：**
```python
# _get_bond_change_pct_from_mysql 函数
from ..config import Config  # 相对导入，指向 dashboard2/config.py
engine = create_engine(Config.MYSQL_URI)  # 报错！Config 没有 MYSQL_URI
```

**修复方案：** 在 `dashboard2/config.py` 中添加 MySQL 配置

```python
# dashboard2/config.py
from gs2026.utils import config_util

class Config:
    # ... 原有配置 ...
    
    # 【新增】MySQL配置
    _config = config_util.load_config()
    _mysql_config = _config.get('mysql', {})
    MYSQL_HOST = _mysql_config.get('host', '192.168.0.101')
    MYSQL_PORT = _mysql_config.get('port', 3306)
    MYSQL_USER = _mysql_config.get('user', 'root')
    MYSQL_PASSWORD = _mysql_config.get('password', '123456')
    MYSQL_DATABASE = _mysql_config.get('database', 'gs')
    MYSQL_URI = f"mysql+pymysql://{MYSQL_USER}:{MYSQL_PASSWORD}@{MYSQL_HOST}:{MYSQL_PORT}/{MYSQL_DATABASE}?charset=utf8"
```

**提交：** `1c14df1` fix: 修复债券实时场景涨跌幅缺失问题

## 根因总结

| 问题 | 根因 | 修复 |
|------|------|------|
| 时间轴无涨跌幅 | `at-time` 接口未调用 `_enrich_bond_data()` | 添加债券数据增强逻辑 |
| 实时无涨跌幅 | `dashboard2/config.py` 缺少 `MYSQL_URI` | 添加 MySQL 配置 |
