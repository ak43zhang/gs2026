## 二、Bug修复记录

### 2.1 语法错误修复

**问题**: `SyntaxError: unexpected character after line continuation character`

**原因**: f-string中直接使用 `\` 转义导致语法错误

**修复**: 使用辅助函数 `escape_value()` 替代内嵌转义

```python
# 修复前
sql = f"INSERT ... VALUES ('{value.replace("'", "''")}')"

# 修复后
def escape_value(v):
    if v is None:
        return 'NULL'
    return "'" + str(v).replace("'", "''") + "'"
```

**影响**: 4个MySQL保存函数（news/domain/ztb/notice）

### 2.2 枚举字段验证

**问题**: `1265 (01000): Data truncated for column 'risk_level' at row 1`

**原因**: AI返回的数据包含空格或非法值

**修复**: 添加数据清洗和默认值

```python
risk_level = str(risk_level).strip()
if risk_level not in ['高', '中', '低']:
    risk_level = '中'  # 默认值
```

### 2.3 涨停时间解析错误

**问题**: `invalid literal for int() with base 10: '2026-04-13 09'`

**原因**: AI返回的时间格式为 `2026-04-13 09:30:00`，但代码期望 `09:30:00`

**修复**: 提取时间部分

```python
if ' ' in zt_time:
    zt_time = zt_time.split(' ')[1]  # 提取 '09:30:00'
```

### 2.4 涨停时间格式转换

**问题**: 前端显示为空

**原因**: MySQL TIME类型通过pandas读取后变成Timedelta对象

**修复**: 服务层转换为字符串

```python
if hasattr(item['zt_time'], 'total_seconds'):
    total_seconds = int(item['zt_time'].total_seconds())
    hours = total_seconds // 3600
    minutes = (total_seconds % 3600) // 60
    seconds = total_seconds % 60
    item['zt_time'] = f"{hours:02d}:{minutes:02d}:{seconds:02d}"
```
