## 一、现状诊断

### 1.1 当前评分逻辑

```python
# result_processor.py → _extract_notice_record()
risk_map  = {'高': 75, '中': 50, '低': 25}   # 影响力度 → risk_score
type_map  = {'利好': 75, '中性': 50, '利空': 25}  # 消息类型 → type_score
```

仅根据1个维度做固定映射，每个评分只有3个离散值（25/50/75）。

### 1.2 数据分布（总量154,551条）

| 组合 | 平均分 | 数量 | 占比 |
|------|--------|------|------|
| 低+中性 | 38 | 103,498 | **67.0%** |
| 中+中性 | 50 | 25,338 | 16.4% |
| 低+利好 | 50 | 8,261 | 5.3% |
| 高+利空 | 50 | 5,731 | 3.7% |
| 中+利空 | 38 | 5,350 | 3.5% |
| 中+利好 | 62 | 3,529 | 2.3% |
| 高+中性 | 62 | 2,366 | 1.5% |
| 高+利好 | 75 | 286 | 0.2% |
| 低+利空 | 25 | 192 | 0.1% |

### 1.3 三大致命问题

#### 问题1：67%数据堆积在同一分数，零区分度

超过2/3的公告得分都是38（低+中性），完全无法筛选出值得隔夜的标的。

#### 问题2：积极vs保守评分倒挂

| 隔夜态度 | 数量 | 风险分 | 类型分 | 评分范围 |
|----------|------|--------|--------|---------|
| 积极（应该隔夜） | 439 | 33.3 | 48.7 | 25-75 |
| 保守（应该回避） | 113 | 43.1 | 34.7 | 25-50 |
| 中性 | 448 | 29.5 | 53.0 | 25-75 |

本应隔夜的"积极"组，风险分(33)反而低于"保守"组(43)。评分不仅没有区分度，方向还是反的。

#### 问题3：忽略隔夜超短最关键的维度

| 已有字段 | 对隔夜超短的价值 | 是否参与评分 |
|----------|----------------|-------------|
| open_prediction 开盘预判 | ⭐⭐⭐⭐⭐ 最核心 | ❌ 未使用 |
| market_expectation 市场预期 | ⭐⭐⭐⭐ 放大波动 | ❌ 未使用 |
| duration 持续性 | ⭐⭐⭐ 一日游不值得隔夜 | ❌ 未使用 |
| risk_level 影响力度 | ⭐⭐⭐ 基础过滤 | ✅ 映射过粗 |
| notice_type 消息类型 | ⭐⭐ 方向参考 | ✅ 利空=25不合理 |

### 1.4 评分档位与消息类型分布

| 档位 | 数量 | 利好 | 利空 | 利空占比 |
|------|------|------|------|---------|
| 高分档(≥60) | 6,181 | 3,815 | 0 | 0.0% |
| 中分档(40-60) | 39,330 | 8,261 | 5,731 | 14.6% |
| 低分档(<40) | 109,040 | 0 | 5,542 | 5.1% |

高分档中0条利空（因为利空=25，永远进不了高分档），这对隔夜超短是严重缺陷——强利空信号同样具有高操作价值。

## 二、优化方案

### 2.1 核心思路

- **保留** `risk_score` + `type_score`（向后兼容，优化映射逻辑）
- **新增** `overnight_score`（隔夜综合评分 0-100），专为隔夜超短决策优化

### 2.2 新增字段

```sql
ALTER TABLE analysis_notice_detail_2026 
  ADD COLUMN overnight_score TINYINT UNSIGNED DEFAULT 0 
  COMMENT '隔夜综合评分(0-100)' AFTER type_score;
```

### 2.3 overnight_score 评分公式

```
overnight_score = 开盘预判分(40%) + 影响力度分(25%) + 市场预期分(20%) + 持续性分(15%)
```

#### ① 开盘预判分（权重40%，满分40）

| 开盘预判 | 得分 | 逻辑 |
|----------|------|------|
| 大幅高开(>5%) | 40 | 隔夜做多最佳信号 |
| 大幅低开(>5%) | 38 | 隔夜做空/回避强信号 |
| 高开(2-5%) | 30 | 有隔夜价值 |
| 低开(2-5%) | 28 | 方向明确，有操作价值 |
| 小幅高开(0-2%) | 18 | 幅度有限，性价比低 |
| 小幅低开(0-2%) | 15 | 幅度有限 |
| 平开 | 5 | 隔夜无意义 |
| 空值/未知 | 5 | 默认最低 |

> 关键设计：大幅低开也给高分（38），因为方向明确本身就有隔夜价值。隔夜超短需要的是"波动"而非"方向"。

#### ② 影响力度分（权重25%，满分25）

| 影响力度 | 得分 |
|----------|------|
| 高 | 25 |
| 中 | 12 |
| 低 | 3 |

#### ③ 市场预期分（权重20%，满分20）

| 市场预期 | 得分 |
|----------|------|
| 超预期 | 20 |
| 低于预期 | 18 |
| 符合预期 | 8 |
| 空值/未知 | 5 |

#### ④ 持续性分（权重15%，满分15）

| 持续性 | 得分 |
|----------|------|
| 2-3日 | 15 |
| 一周以上 | 13 |
| 持续发酵 | 12 |
| 一日游 | 6 |
| 空值/未知 | 5 |

### 2.4 评分档位对应操作建议

| 档位 | 分数 | 含义 | 操作建议 |
|------|------|------|---------|
| S级 | 85-100 | 极强隔夜信号 | 重仓隔夜（配合方向判断） |
| A级 | 70-84 | 强隔夜信号 | 可积极隔夜 |
| B级 | 50-69 | 一般信号 | 轻仓试探或观望 |
| C级 | 30-49 | 弱信号 | 不建议隔夜 |
| D级 | 0-29 | 无隔夜价值 | 完全回避 |

### 2.5 risk_score 优化

```python
# 旧逻辑：3个固定值
risk_map = {'高': 75, '中': 50, '低': 25}

# 新逻辑：影响力度 × 市场预期 组合
def calc_risk_score(risk_level, market_expectation):
    base = {'高': 70, '中': 45, '低': 20}.get(risk_level, 45)
    bonus = {'超预期': 20, '低于预期': 15, '符合预期': 5}.get(market_expectation, 0)
    return min(base + bonus, 99)
```

| 组合 | 新评分 |
|------|--------|
| 高+超预期 | 90 |
| 高+低于预期 | 85 |
| 高+符合预期 | 75 |
| 中+超预期 | 65 |
| 中+符合预期 | 50 |
| 低+任何 | ≤40 |

### 2.6 type_score 优化

```python
# 旧逻辑：利空永远=25
type_map = {'利好': 75, '中性': 50, '利空': 25}

# 新逻辑：方向强度评分（利空也可以得高分）
def calc_type_score(notice_type, open_prediction):
    gap_score = _parse_gap_level(open_prediction)  # 0-4
    direction = {'利好': 1, '利空': -1, '中性': 0}.get(notice_type, 0)
    clarity = abs(direction) * 30 + gap_score * 10 + 20
    return min(int(clarity), 99)
```
