## 1. 需求概述

### 1.1 背景
新闻分析当前按自然日展示数据，但用户期望按**交易日时间范围**展示，确保数据连续性。

### 1.2 目标
- 当前时间模式：展示上一个交易日收盘到当前时间的新闻
- 日期选择模式：展示所选日期上一个交易日收盘到该日期24点的新闻
- 短时间补偿：时间范围低于6小时时，扩展到上上个交易日

---

## 2. 时间范围定义

### 2.1 当前时间模式（无日期选择）

```
范围：上一个交易日 15:00 → 当前时间

例外情况（时间范围 < 6小时）：
范围：上上个交易日 15:00 → 当前时间
```

**示例**：
| 当前时间 | 上个交易日 | 时间范围 | 是否扩展 |
|---------|-----------|---------|---------|
| 4月15日 10:00 | 4月14日 | 19小时 | 否 |
| 4月15日 16:00 | 4月14日 | 25小时 | 否 |
| 4月15日 20:00（数据少）| 4月14日 | 5小时 | 是 → 4月13日15:00 |

### 2.2 日期选择器模式

```
范围：所选日期的上一个交易日 15:00 → 所选日期 23:59:59
```

**示例**：
| 选择日期 | 上个交易日 | 时间范围 |
|---------|-----------|---------|
| 4月15日（周二）| 4月14日（周一）| 4/14 15:00 ~ 4/15 24:00 |
| 4月14日（周一）| 4月11日（周五）| 4/11 15:00 ~ 4/14 24:00 |
| 4月12日（周六）| 4月11日（周五）| 4/11 15:00 ~ 4/12 24:00 |

## 3. 核心算法

### 3.1 时间范围计算

```python
def get_news_time_range(target_date=None, target_time=None):
    """
    计算新闻分析的时间范围
    
    Args:
        target_date: 选择的日期（None表示当前时间模式）
        target_time: 当前时间（仅当前时间模式使用）
    
    Returns:
        dict: {
            'start_time': datetime,      # 范围开始
            'end_time': datetime,        # 范围结束
            'display_date': date,        # 显示日期
            'trading_day': date,         # 参考交易日
            'is_extended': bool,         # 是否已扩展
            'hours_span': float          # 时间跨度（小时）
        }
    """
    trading_days = get_trading_days_from_jyrl()
    
    if target_date is None:
        # 当前时间模式
        now = target_time or datetime.now()
        today = now.date()
        
        # 获取上个交易日
        prev_trading_day = get_previous_trading_day(today, trading_days)
        start_time = datetime.combine(prev_trading_day, time(15, 0, 0))
        end_time = now
        
        # 检查是否需要扩展
        hours_span = (end_time - start_time).total_seconds() / 3600
        is_extended = False
        
        if hours_span < 6:
            # 扩展到上上个交易日
            prev_prev_trading_day = get_previous_trading_day(prev_trading_day, trading_days)
            start_time = datetime.combine(prev_prev_trading_day, time(15, 0, 0))
            hours_span = (end_time - start_time).total_seconds() / 3600
            is_extended = True
            trading_day = prev_prev_trading_day
        else:
            trading_day = prev_trading_day
            
        display_date = prev_trading_day
        
    else:
        # 日期选择器模式
        prev_trading_day = get_previous_trading_day(target_date, trading_days)
        
        start_time = datetime.combine(prev_trading_day, time(15, 0, 0))
        end_time = datetime.combine(target_date, time(23, 59, 59))
        
        hours_span = (end_time - start_time).total_seconds() / 3600
        is_extended = False
        trading_day = target_date
        display_date = target_date
    
    return {
        'start_time': start_time,
        'end_time': end_time,
        'display_date': display_date,
        'trading_day': trading_day,
        'is_extended': is_extended,
        'hours_span': round(hours_span, 1)
    }
```

### 3.2 交易日获取

```python
def get_previous_trading_day(date, trading_days):
    """获取指定日期的上一个交易日"""
    sorted_days = sorted([d for d in trading_days if d < date], reverse=True)
    return sorted_days[0] if sorted_days else date - timedelta(days=1)

def get_trading_days_from_jyrl():
    """从data_jyrl表获取交易日历"""
    sql = "SELECT DISTINCT date FROM data_jyrl ORDER BY date"
    df = pd.read_sql(sql, db_engine)
    return set(df['date'].tolist())
```

## 4. 数据库查询修改

### 4.1 新闻列表查询

**修改前**：
```sql
SELECT * FROM analysis_news_detail_2026 
WHERE news_date = '2026-04-15'
ORDER BY publish_time DESC
LIMIT 20 OFFSET 0
```

**修改后**：
```sql
SELECT * FROM analysis_news_detail_2026 
WHERE publish_time >= '2026-04-14 15:00:00' 
  AND publish_time <= '2026-04-15 10:00:00'
ORDER BY publish_time DESC
LIMIT 20 OFFSET 0
```

## 7. 边界情况处理

| 场景 | 处理方案 |
|------|---------|
| 当前时间是周末 | 上个交易日为周五，正常计算 |
| 选择节假日 | 找该日期的上个交易日，范围到所选日期24点 |
| 6小时扩展后仍不足 | 不再继续扩展，保持上上个交易日范围 |
| 无交易日历数据 | 回退：昨天15:00 ~ 今天24:00 |
| 数据库无数据 | 显示空列表，提示"该时间段暂无数据" |
