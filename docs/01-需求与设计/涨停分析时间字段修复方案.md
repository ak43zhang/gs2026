# 涨停分析时间字段修复方案

> 日期：2026-04-14  
> 问题：analysis_ztb_detail_2026表zt_time不准确，需从ztb_day获取首次涨停时间  
> 方案：方案A（在result_processor中查询ztb_day）

---

## 一、问题描述

### 1.1 当前问题
- `analysis_ztb_detail_2026`表中的`zt_time`字段来源于AI分析结果
- AI分析的涨停时间可能不准确
- 需要使用`ztb_day`表中的真实首次涨停时间

### 1.2 数据来源
| 表名 | 字段 | 说明 |
|------|------|------|
| ztb_day | 首次涨停时间 | 真实涨停时间，可信度高 |
| analysis_ztb_detail_2026 | zt_time | 当前为AI分析时间，需替换 |

---

## 二、解决方案（方案A）

### 2.1 设计思路
在`result_processor.py`的`_extract_ztb_record`函数中：
1. 接收`stock_code`和`trade_date`参数
2. 查询`ztb_day`表获取首次涨停时间
3. 使用真实时间替换AI分析时间
4. **优化**：单次分析批次内缓存查询结果，避免重复查询

### 2.2 代码修改点

#### 修改1：添加查询函数
```python
def _get_zt_time_from_db(stock_code: str, trade_date: str) -> Optional[str]:
    """从ztb_day查询首次涨停时间"""
    try:
        sql = f"""
            SELECT 首次涨停时间 
            FROM ztb_day 
            WHERE 股票代码 = '{stock_code}' 
            AND trade_date = '{trade_date}'
            LIMIT 1
        """
        df = pd.read_sql(sql, engine)
        if not df.empty:
            # 处理时间格式，可能是Timedelta或字符串
            zt_time = df.iloc[0]['首次涨停时间']
            if hasattr(zt_time, 'total_seconds'):
                # Timedelta类型
                total_seconds = int(zt_time.total_seconds())
                hours = total_seconds // 3600
                minutes = (total_seconds % 3600) // 60
                seconds = total_seconds % 60
                return f"{hours:02d}:{minutes:02d}:{seconds:02d}"
            else:
                return str(zt_time)
    except Exception as e:
        logger.warning(f"查询ztb_day失败: {e}")
    return None
```

#### 查询缓存机制
为避免同一批次内重复查询，使用函数内缓存：

```python
# 模块级缓存，单次分析有效
_zt_time_cache: Dict[str, str] = {}

def _get_zt_time_cached(stock_code: str, trade_date: str) -> Optional[str]:
    """带缓存的涨停时间查询"""
    cache_key = f"{stock_code}_{trade_date}"
    if cache_key in _zt_time_cache:
        return _zt_time_cache[cache_key]
    
    zt_time = _get_zt_time_from_db(stock_code, trade_date)
    if zt_time:
        _zt_time_cache[cache_key] = zt_time
```
