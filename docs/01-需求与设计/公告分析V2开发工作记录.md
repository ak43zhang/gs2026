## 一、评分体系升级

### 1.1 问题诊断

- 原评分仅3个离散值（25/50/75），67%数据堆积在38分
- 积极vs保守态度评分倒挂（积极33 < 保守43）
- 忽略开盘预判、市场预期、持续性等关键维度
- 利空永远得25分，无法识别强利空信号的隔夜价值

### 1.2 实施内容

| 变更 | 说明 |
|------|------|
| `risk_score` 优化 | 影响力度 × 市场预期 组合计算（0-99连续值） |
| `type_score` 优化 | 方向强度 × 开盘幅度 组合计算（利空也能得高分） |
| `overnight_score` 新增 | 隔夜综合评分（0-100），4维度加权：开盘预判40% + 影响力度25% + 市场预期20% + 持续性15% |
| DB变更 | `analysis_notice_detail_2026` 新增 `overnight_score TINYINT` |
| 代码变更 | `result_processor.py` → `_extract_notice_record()` 全面重写评分逻辑 |

### 1.3 测试验证

| 场景 | risk | type | overnight | 档位 |
|------|------|------|-----------|------|
| 高+利好+超预期+大幅高开+一周以上 | 90 | 90 | 98 | S级 |
| 高+利空+低于预期+大幅低开+一周以上 | 85 | 90 | 94 | S级 |
| 中+利好+符合预期+小幅高开+2-3日 | 50 | 80 | 65 | B级 |
| 高+利空+符合预期+平开+一日游 | 75 | 50 | 44 | C级 |
| 低+中性+空值 | 20 | 20 | 18 | D级 |

## 三、公告原文抓取工具

### 3.1 需求

从东方财富API获取公告全文，写入 `jhsaggg2026.公告原文` 字段。

### 3.2 技术方案

- API端点: `https://np-cnotice-stock.eastmoney.com/api/content/ann?art_code={art_code}`
- 从 `网址` 字段提取 `art_code`（正则 `(AN\d+)\.html`）
- 返回字段: `data.notice_content`（纯文本公告全文）

### 3.3 实施内容

| 变更 | 说明 |
|------|------|
| 新建文件 | `src/gs2026/collection/risk/notice_content_fetcher.py` |
| DB变更 | `jhsaggg2026` 新增 `公告原文 LONGTEXT` + `content_status TINYINT` |
| 配置 | `configs/settings.yaml` 新增 `notice_content_fetcher` 配置块 |
| 集成 | `notice_risk_history.py` 新增2处（import + try调用，开关控制） |

### 3.4 配置参数

```yaml
notice_content_fetcher:
    enabled: false              # 采集时同步抓取开关（默认关闭）
    api_url: "https://np-cnotice-stock.eastmoney.com/api/content/ann"
    delay_min: 0.3
    delay_max: 0.8
    request_timeout: 10
    max_consecutive_fail: 10
    pause_seconds: 60
    max_retries: 3
    batch_size: 2000
    table_prefix: "jhsaggg"
    backfill_start: "2026-04-01"
    backfill_end: "2026-05-02"
```

### 3.5 使用方式

```bash
# 独立补跑（读取settings.yaml中的时间配置）
.venv\Scripts\python.exe -m gs2026.collection.risk.notice_content_fetcher

# 命令行指定时间段
.venv\Scripts\python.exe -m gs2026.collection.risk.notice_content_fetcher --start 2026-04-01 --end 2026-05-02

# 单日抓取
.venv\Scripts\python.exe -m gs2026.collection.risk.notice_content_fetcher --date 2026-04-30
```

### 3.6 年份分表处理

- 配置 `table_prefix: "jhsaggg"`，程序自动按日期年份拼接表名
- 跨年范围（如 2025-12-01 ~ 2026-05-02）自动分别查询 `jhsaggg2025` 和 `jhsaggg2026`

### 3.7 测试结果

| 指标 | 结果 |
|------|------|
| 测试量 | 100条（2026-04-30） |
| 有内容 | 98条 |
| 仅PDF | 3条 |
| 失败 | 0条 |
| 耗时 | ~2分钟 |

### 3.8 content_status 状态码

| 值 | 含义 |
|----|------|
| 0 | 未抓取（默认） |
| 1 | 已抓取有内容 |
| 2 | 无内容（仅PDF） |
| 3 | 抓取失败 |
