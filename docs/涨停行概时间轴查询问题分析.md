# 涨停行概时间轴查询问题分析

## 问题描述

时间轴模式点击查询没有数据，但默认模式有数据。

## 对比分析

### 默认模式 (`get_ztb_tags`)

```python
def get_ztb_tags(date: str = None) -> dict:
    # 1. 获取涨停股票代码（当天）
    #    - 优先Redis: redis_util.get_zt_stocks_from_redis(date, 'stock')
    #    - 回退MySQL: SELECT DISTINCT stock_code FROM monitor_gp_sssj_{date} WHERE is_zt = 1
    
    # 2. 获取涨停股票代码（历史日期）
    #    - 从analysis_ztb_detail_{year}表查询
    
    # 3. 从宽表内存缓存获取行业和概念
    #    - 使用_stock_cache
    
    # 4. 统计并返回
```

**关键特点：**
- 使用 `redis_util.get_zt_stocks_from_redis(date, 'stock')` 获取当天涨停
- 历史日期使用 `analysis_ztb_detail_{year}` 表
- 查询条件是 `is_zt = 1`，没有时间限制（获取全天所有涨停）

### 时间轴模式 (`get_ztb_snapshot`)

```python
def get_ztb_snapshot(date: str, time: str) -> Dict:
    # 1. 获取指定时间点的涨停股票
    #    - Redis: monitor_gp_sssj_{date_str}:{time}
    #    - MySQL: SELECT stock_code FROM monitor_gp_sssj_{date} WHERE time = '{time}' AND is_zt = 1
    
    # 2. 分析行业和概念（复用逻辑）
```

**关键特点：**
- 必须指定时间点 `time`
- 查询特定时间点的涨停数据
- 如果该时间点没有涨停数据，返回空

## 问题原因

### 可能原因1：时间点没有涨停数据

`monitor_gp_sssj`表存储的是实时数据，每个时间点只存储当时的股票状态。

**问题：**
- 如果选择的 `time` 点没有涨停股票，`is_zt = 1` 的记录数为0
- 默认模式查询全天所有 `is_zt = 1` 的记录，不受时间限制

**示例：**
- 默认模式：查询 `2026-04-29` 全天所有涨停股票 → 返回100只
- 时间轴模式：查询 `2026-04-29 09:30:00` 的涨停股票 → 可能只有5只（开盘涨停）
- 时间轴模式：查询 `2026-04-29 09:30:15` 的涨停股票 → 可能有10只

### 可能原因2：Redis key格式不匹配

时间轴查询使用的Redis key：
```
monitor_gp_sssj_20260429:09:30:00
```

但实际存储的key可能不同（需要确认）。

### 可能原因3：时间格式不匹配

前端传递的时间格式与Redis/MySQL存储的格式不一致。

## 解决方案

### 方案A：沿用默认查询方式（推荐）

**核心思路：** 时间轴模式也应该查询全天涨停数据，而不是特定时间点。

**修改逻辑：**
```python
def get_ztb_snapshot(date: str, time: str) -> Dict:
    """获取指定时间点的涨停快照
    
    逻辑修改：
    - 不是查询该时间点的涨停股票
    - 而是查询到该时间点为止，所有曾经涨停的股票
    - 展示这些股票在该时间点的状态
    """
    date_str = date.replace('-', '')
    
    # 1. 获取到该时间点为止的所有涨停股票（沿用默认逻辑）
    #    - 使用ever_zt字段或查询历史涨停表
    
    # 2. 获取这些股票在该时间点的实时数据
    
    # 3. 分析行业和概念
```

**具体实施：**

1. **修改数据查询逻辑**
   - 查询 `monitor_gp_sssj_{date}` 表中 `ever_zt = 1` 且 `time <= '{time}'` 的最新记录
   - 或者查询 `analysis_ztb_detail_{year}` 表获取历史涨停

2. **修改 `_get_zt_stocks_at_time` 函数**
   ```python
   def _get_zt_stocks_at_time(date_str: str, time: str) -> List[str]:
       """获取到指定时间点为止的所有涨停股票"""
       # 查询ever_zt=1且time<=指定时间的股票
       sql = f"""
           SELECT DISTINCT stock_code FROM {table_name}
           WHERE ever_zt = 1 AND time <= '{time}'
       """
       # 或者查询历史涨停表
   ```

### 方案B：修复时间点查询

如果确实需要查询特定时间点的涨停，需要确认：
1. Redis中是否存在该时间点的数据
2. 时间格式是否正确
3. 该时间点是否真的有涨停股票

## 建议

**推荐方案A**，原因：
1. 与默认模式逻辑一致（查询涨停股票，不限制时间点）
2. 用户期望看到"到该时间点为止"的涨停情况
3. 数据更完整，体验更好
