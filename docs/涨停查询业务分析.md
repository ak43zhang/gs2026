# 涨停查询业务分析

## 字段含义

### `is_zt` vs `ever_zt`

| 字段 | 含义 | 示例 |
|------|------|------|
| `is_zt` | **当前时刻是否涨停** | 09:30:00 is_zt=1, 09:30:15 is_zt=0（打开涨停） |
| `ever_zt` | **当天是否曾涨停** | 只要当天曾涨停，所有记录的ever_zt=1 |

## 两种查询方式的区别

### 方式A：`is_zt=1 AND time='指定时间'`

**含义：** 查询"指定时间点正在涨停"的股票

**适用场景：**
- 查看某个瞬间的涨停情况
- 了解当时市场上有哪些股票在涨停板上

**示例：**
- 09:30:00 查询 → 返回10只（当时正在涨停）
- 09:30:15 查询 → 返回8只（2只打开涨停）
- 14:30:00 查询 → 返回50只（下午涨停潮）

### 方式B：`ever_zt=1 AND time<='指定时间'`

**含义：** 查询"到指定时间点为止曾涨停"的股票

**适用场景：**
- 复盘分析：查看当天哪些股票曾涨停
- 统计涨停持续性

**示例：**
- 09:30:00 查询 → 返回10只
- 09:30:15 查询 → 返回10只（包含已打开的）
- 14:30:00 查询 → 返回100只（全天累计）

## 当前实现的问题

### 现状
- `get_ztb_snapshot` 使用 `ever_zt=1` → 获取曾涨停股票
- `filter_ztb_snapshot` 筛选时展示的是这些股票的**当前状态**

### 问题
用户期望的是：**查看指定时间点的瞬时涨停状态**

即：
1. 选择 09:30:00 → 看到当时正在涨停的股票及其当时的价格
2. 选择 14:30:00 → 看到当时正在涨停的股票及其当时的价格

## 正确的业务逻辑

### 场景：复盘指定时间点的涨停情况

**用户需求：** "我想看看14:30的时候，市场上有哪些股票在涨停，当时的价格是多少"

**正确实现：**
```sql
-- 查询指定时间点正在涨停的股票
SELECT * FROM monitor_gp_sssj_{date} 
WHERE time = '14:30:00' AND is_zt = 1

-- 获取这些股票的行业概念（从宽表）
-- 展示当时的涨跌幅、价格等
```

### 场景：查看当天所有曾涨停的股票

**用户需求：** "我想看看今天哪些股票曾涨停过"

**正确实现：**
```sql
-- 查询当天曾涨停的所有股票
SELECT DISTINCT stock_code FROM monitor_gp_sssj_{date} 
WHERE ever_zt = 1
```

## 建议方案

### 方案1：时间轴查询 `is_zt=1`（推荐）

**理由：**
1. 时间轴选择"14:30"，用户期望看到的是14:30那一刻的涨停情况
2. 与默认模式（最新时间涨停）逻辑一致，只是时间不同
3. 数据更准确，反映当时市场状态

**修改：**
```python
def _get_zt_stocks_at_time(date_str: str, time: str) -> pd.DataFrame:
    """获取指定时间点正在涨停的股票"""
    # 查询 is_zt=1 且 time=指定时间
    sql = f"""
        SELECT * FROM monitor_gp_sssj_{date_str}
        WHERE time = '{time}' AND is_zt = 1
    """
```

### 方案2：保持 `ever_zt=1`

**理由：**
1. 可以看到全天曾涨停的所有股票
2. 了解哪些股票"曾经强势"

**但：** 与时间轴选择时间点的交互不符

## 结论

**推荐方案1：使用 `is_zt=1 AND time='指定时间'`**

时间轴模式应该查询"指定时间点正在涨停"的股票，而不是"到该时间点为止曾涨停"的股票。

用户选择"14:30"，期望看到的是14:30那一刻的涨停情况，而不是全天累计。
