# 智能选股债券数据源优化

## 版本
- v1.0 (2026-04-21)

## 问题描述

交叉行概选股功能中，债券映射使用 `data_bond_ths`（同花顺）表，该表包含已退市、已到期、已赎回的债券（如博特转债，到期日2026-03-12），导致前端展示已不存在的债券数据。

## 数据分析

### 数据源对比

| 数据源 | 表名 | 记录数 | 说明 |
|--------|------|--------|------|
| 同花顺 | `data_bond_ths` | 919 | 包含已到期290条、未上市5条、已退市/赎回约300条 |
| 集思录 | `data_bond_qs_jsl` | **343** | **仅包含实际在交易的转债** |

### `data_bond_ths` 状态分布

| 状态 | 数量 |
|------|------|
| 未上市（上市日期为空） | 5 |
| 已到期（到期时间 < 今天） | 290 |
| 已上市且未到期 | 624 |

624条"已上市且未到期"仍远超实际交易的300+条，原因是部分转债已强制赎回或转股退市，但同花顺表中未标记状态。

### `data_bond_qs_jsl` 表结构

| 字段 | 类型 | 说明 |
|------|------|------|
| 代码 | text | 债券代码，如 113618 |
| 名称 | text | 债券名称，如 诺泰转债 |
| 现价 | double | 当前价格 |
| 正股代码 | text | **标准6位股票代码**，如 603538 |
| 正股名称 | text | 正股名称，如 诺泰生物 |
| 规模 | double | 发行规模（亿元） |
| 剩余规模 | double | 剩余规模（亿元） |
| 转股起始日 | date | 转股起始日期 |
| 强赎触发价 | double | 强赎触发价格 |
| 到期日 | date | 到期日期 |
| 转股价 | double | 转股价格 |
| 正股价 | double | 正股价格 |
| 强赎价 | double | 强赎价格 |
| 强赎条件描述 | text | 强赎条件说明 |
| 强赎条款 | text | 强赎条款全文 |
| 强赎状态 | text | 已公告强赎 / 未公告 |

### 数据验证

- 集思录正股代码与行业成分股表（`data_industry_code_component_ths`）匹配率：**99.7%**（339/340）
- 集思录数据全部为未到期状态（0条已到期）
- 博特转债（113571，到期日2026-03-12）不在集思录数据中 ✅

## 修改方案

### 修改文件

`src/gs2026/dashboard2/services/stock_picker_service.py`

### 修改内容

#### `warm_up_cache()` 函数中的债券映射

**修改前**（使用 `data_bond_ths`，依赖列索引）：
```python
bond_map = {}
with mysql_tool.engine.connect() as conn:
    rows = pd.read_sql("SELECT * FROM data_bond_ths", conn).to_dict('records')
for row in rows:
    values = list(row.values())
    stock_code = values[12]   # 第12列: 正股代码2
    bond_code = values[1]     # 第1列: 债券代码
    bond_name = values[2]     # 第2列: 债券名称
    if stock_code:
        bond_map[stock_code] = {'code': bond_code, 'name': bond_name}
```

**修改后**（使用 `data_bond_qs_jsl`，按字段名查询）：
```python
bond_map = {}
with mysql_tool.engine.connect() as conn:
    rows = pd.read_sql(
        "SELECT `代码` as bond_code, `名称` as bond_name, "
        "`正股代码` as stock_code FROM data_bond_qs_jsl",
        conn
    ).to_dict('records')
for row in rows:
    stock_code = row['stock_code']
    if stock_code:
        bond_map[stock_code] = {'code': row['bond_code'], 'name': row['bond_name']}
```

### 验证结果

| 指标 | 修改前 | 修改后 |
|------|--------|--------|
| 债券映射数 | 836（含退市） | **339**（仅交易中） |
| 博特转债(113571) | 出现 | 已过滤 |
| 与行业表匹配率 | 99.3% | 99.7% |
| 代码可读性 | 依赖列索引 | 按字段名查询 |
| 电力行业有债券股票 | - | 9只 |

### 优势

1. **数据精准**：集思录数据只包含实际在交易的转债
2. **代码简洁**：直接用字段名查询，不依赖脆弱的列索引
3. **自动更新**：集思录数据定期采集，退市债券自动消失

### 注意事项

- 集思录数据由 `data_bond_qs_jsl` 表的采集任务维护
- 如果集思录采集中断，债券数据可能不完整
- 每次缓存预热时自动从该表读取最新数据
