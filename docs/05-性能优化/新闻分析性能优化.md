# 新闻分析性能优化文档

**优化日期**: 2026-04-15  
**优化目标**: `/api/news/list` 和 `/api/news/stats` 接口响应速度  
**预期效果**: 查询速度提升 50-80%

---

## 一、问题分析

### 1.1 慢查询原因

#### `/api/news/list`
1. **两次查询**: 先执行 `COUNT(*)` 获取总数，再执行数据查询
2. **无索引**: `publish_time`、`news_type`、`news_size` 等字段无索引
3. **Python循环**: 逐行转换数据，开销大

#### `/api/news/stats`
1. **时间范围计算**: 每次调用都重新计算交易日时间范围
2. **聚合查询**: 使用多个 `SUM(CASE WHEN...)`，数据量大时慢

---

## 二、优化方案

### 方案1: 数据库索引优化 ✅ 已实施

#### 添加的索引

```sql
-- 1. 时间范围查询索引（最重要）
ALTER TABLE analysis_news_detail_2026 ADD INDEX idx_publish_time (publish_time);

-- 2. 消息类型索引
ALTER TABLE analysis_news_detail_2026 ADD INDEX idx_news_type (news_type);

-- 3. 消息大小索引
ALTER TABLE analysis_news_detail_2026 ADD INDEX idx_news_size (news_size);

-- 4. 复合索引（最常用组合）
ALTER TABLE analysis_news_detail_2026 ADD INDEX idx_time_type_size (publish_time, news_type, news_size);

-- 5. 时间+评分复合索引（排序优化）
ALTER TABLE analysis_news_detail_2026 ADD INDEX idx_time_score (publish_time, composite_score);
```

#### 索引效果
- `idx_publish_time`: 优化时间范围查询 `WHERE publish_time BETWEEN 'xxx' AND 'yyy'`
- `idx_time_type_size`: 优化组合筛选 `WHERE publish_time BETWEEN ... AND news_type='利好' AND news_size='重大'`
- `idx_time_score`: 优化按评分排序 `ORDER BY composite_score DESC`

### 方案3: 查询逻辑优化 ✅ 已实施

#### 优化1: 合并计数和查询

**优化前**:
```python
# 两次查询
count_sql = f"SELECT COUNT(*) as cnt FROM ... WHERE {where_str}"
count_df = pd.read_sql(count_sql, conn)
total = int(count_df.iloc[0]['cnt'])

data_sql = f"SELECT ... FROM ... WHERE {where_str} LIMIT ..."
df = pd.read_sql(data_sql, conn)
```

**优化后**:
```python
# 一次查询（使用 SQL_CALC_FOUND_ROWS）
data_sql = f"""SELECT SQL_CALC_FOUND_ROWS ... FROM ... WHERE {where_str} LIMIT ..."""
df = pd.read_sql(data_sql, conn)

# 在同一个连接中获取总数
total_df = pd.read_sql("SELECT FOUND_ROWS() as total", conn)
total = int(total_df.iloc[0]['total'])
```

**效果**: 减少一次查询，减少网络往返

#### 优化2: 批量数据转换

**优化前**:
```python
items = []
for _, row in df.iterrows():
    item = row.to_dict()
    # 逐个字段转换
    for key in ('publish_time', 'analysis_time'):
        if item.get(key) is not None:
            item[key] = str(item[key])
    # 逐个解析JSON
    for key in ('sectors', 'concepts', 'leading_stocks'):
        try:
            val = item.get(key)
            item[key] = json.loads(val) if isinstance(val, str) else []
        except:
            item[key] = []
    items.append(item)
```

**优化后**:
```python
# 批量转换时间字段
for key in ('publish_time', 'analysis_time'):
    if key in df.columns:
        df[key] = df[key].astype(str)

# 批量解析JSON
for field in ['sectors', 'concepts', 'leading_stocks', 'sector_details']:
    if field in df.columns:
        df[field] = df[field].apply(lambda x: json.loads(x) if isinstance(x, str) and x else [])

# 批量转换整数
for field in ['importance_score', 'business_impact_score', 'composite_score']:
    if field in df.columns:
        df[field] = df[field].fillna(0).astype(int)

# 一次性转换为字典列表
items = df.to_dict('records')
```

**效果**: 使用 pandas 向量化操作，减少Python循环开销

## 四、后续优化建议

### 方案2: 查询结果缓存（待实施）

使用 Redis 缓存查询结果（5-10秒）:

```python
def get_news_list(...):
    # 生成缓存key
    cache_key = f"news:list:{date}:{news_type}:{news_size}:{page}:{page_size}:{sort_by}"
    
    # 先查缓存
    cached = redis.get(cache_key)
    if cached:
        return json.loads(cached)
    
    # 执行查询
    result = _get_list_from_mysql_by_time_range(...)
    
    # 缓存结果（10秒）
    redis.setex(cache_key, 10, json.dumps(result))
    return result
```

**预期效果**: 热点数据查询速度提升 80-90%

### 方案4: 分页优化（大数据量时）

使用游标分页代替 OFFSET:

```sql
-- 传统分页（大数据量时慢）
LIMIT 20 OFFSET 10000

-- 游标分页（快）
WHERE publish_time < '上次最后一条的时间'
LIMIT 20
```
