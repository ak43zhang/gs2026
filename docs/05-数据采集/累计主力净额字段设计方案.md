# 累计主力净额字段设计方案

## 背景
当前主力净额 `main_net_amount` 是单条 tick 的净额，涨停后价格不变导致大量记录为0。需要添加累计字段 `cumulative_main_net` 来展示全天累计值。

## 方案B实施细节

### 1. 数据库表结构修改

#### 新增字段
```sql
ALTER TABLE monitor_gp_sssj_20260428
ADD COLUMN cumulative_main_net DECIMAL(15,2) DEFAULT 0 COMMENT '累计主力净额（元）';
```

### 2. 计算逻辑

#### 流程
```
1. 获取上一时刻该股票的累计主力净额
   SELECT cumulative_main_net 
   FROM monitor_gp_sssj_{date} 
   WHERE stock_code = '{code}' AND time < '{current_time}'
   ORDER BY time DESC LIMIT 1

2. 计算当前 tick 的主力净额（Tick价格变化法）
   current_main_net = delta_amount * direction * participation * confidence

3. 计算新的累计值
   new_cumulative = previous_cumulative + current_main_net

4. 保存到 cumulative_main_net 字段
```

### 3. 代码修改位置

#### monitor_stock.py
1. **修改 `calculate_main_force_net_amount` 函数**

### 4. 数据迁移

对于历史数据，需要填充累计值：
```sql
-- 逐只股票计算累计值
UPDATE monitor_gp_sssj_{date} t1
JOIN (
    SELECT 
        stock_code,
        time,
        SUM(main_net_amount) OVER (
            PARTITION BY stock_code 
            ORDER BY time 
            ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
        ) as cumulative_main_net
    FROM monitor_gp_sssj_{date}
) t2 ON t1.stock_code = t2.stock_code AND t1.time = t2.time
SET t1.cumulative_main_net = t2.cumulative_main_net;
```

### 5. 前端展示

股票上攻排行的主力净额字段直接显示 `cumulative_main_net`：
- 单位：万
- 保留1位小数
- 颜色：红=正，绿=负

### 6. 优势

1. **查询速度快** - 直接读取字段，无需实时 SUM
2. **实时累计** - 每个 tick 都知道累计值
3. **支持时间轴** - 可以查看任意时间点的累计值
4. **涨停股票正常显示** - 累计值持续增长
