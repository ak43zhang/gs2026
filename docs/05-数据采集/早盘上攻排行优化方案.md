# 早盘上攻排行计算优化方案

## 当前问题
- WINDOW_SECONDS = 15秒固定间隔
- 9:30:15之前没有15秒前的数据，导致无法计算上攻排行

## 期望行为
- 9:30:00-9:30:15期间：用当前数据与**竞价最后一条记录**对比
- 9:30:15之后：恢复正常15秒区间计算

## 实现方案

### 1. 修改 monitor_stock.py

在 `deal_gp_works` 函数中，早盘特殊时段加载竞价数据：

```python
# ========== 修改：早盘特殊处理 ==========
# 判断是否在9:30:00-9:30:15之间
time_obj = loop_start.time()
is_early_morning = (dt_time(9, 30, 0) <= time_obj < dt_time(9, 30, 15))

if is_auction:
    df_prev = None
    logger.info(f"[集合竞价] {time_full} 跳过前30秒数据计算")
elif is_early_morning:
    # 早盘9:30:00-9:30:15：加载竞价最后一条记录
    df_prev = redis_util.load_dataframe_by_key(sssj_table, key="09:30:00", use_compression=False)
    if df_prev is None or df_prev.empty:
        logger.warning(f"[早盘] {time_full} 无法加载竞价数据(09:30:00)，跳过计算")
        df_prev = None
    else:
        logger.info(f"[早盘] {time_full} 使用竞价数据(09:30:00)作为基准，共{len(df_prev)}条")
else:
    # 正常15秒区间
    window_seconds_offset = (WINDOW_SECONDS + INTERVAL - 1) // INTERVAL
    df_prev = redis_util.load_dataframe_by_offset(sssj_table, offset=window_seconds_offset, use_compression=False)
```
