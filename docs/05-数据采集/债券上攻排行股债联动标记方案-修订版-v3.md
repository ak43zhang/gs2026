## 1. 问题分析

### 1.1 当前数据流
```
┌─────────────────┐     ┌──────────────────┐     ┌─────────────────┐
│ stock_picker    │────▶│  _stock_cache    │────▶│  股票上攻排行    │
│ _service        │     │  {stock_code:    │     │  （缺少bond_code）│
│                 │     │   bond_code}     │     │                 │
└─────────────────┘     └──────────────────┘     └─────────────────┘
```

### 1.2 问题确认
- `stock_picker_service.py` 有 `_stock_cache`，包含 `bond_code` 字段
- `data_service.py` 的 `get_stock_ranking()` 返回的数据**缺少** `bond_code`
- 前端期望 `item.bond_code` 字段用于显示

### 1.3 解决方案
修改 `data_service.py`，在返回股票上攻排行时，从 `_stock_cache` 补充 `bond_code` 和 `bond_name`。

## 3. 债券上攻排行标记逻辑

### 3.1 新思路
**不修改后端存储，仅在前端展示时做关联标记**

```
前端展示债券上攻排行时：
1. 获取当前股票上攻排行数据（已包含bond_code）
2. 提取股票排行中的bond_code列表
3. 遍历债券上攻排行数据
4. 如果债券的code在股票排行的bond_code列表中，添加标记
```

### 3.2 前端实现方案

```javascript
// 在renderRanking函数中（债券排行部分）

async function renderBondRankingWithLink(data) {
    // 1. 获取股票上攻排行数据
    const stockResponse = await fetch('/api/monitor/attack-ranking/stock?limit=60');
    const stockData = await stockResponse.json();
    
    // 2. 提取股票排行中的债券代码集合
    const stockBondCodes = new Set();
    if (stockData.success && stockData.data) {
        stockData.data.forEach(item => {
            if (item.bond_code && item.bond_code !== '-') {
                stockBondCodes.add(item.bond_code);
            }
        });
    }
    
    // 3. 遍历债券数据，添加标记
    data.forEach(item => {
        if (stockBondCodes.has(item.code)) {
            item.is_stock_linked = true;
            item.code_display = `★${item.code}`;
        } else {
            item.is_stock_linked = false;
            item.code_display = item.code;
        }
    });
    
    // 4. 渲染
    renderRanking('bond-ranking', data);
}
```

### 3.3 优缺点

| 方案 | 优点 | 缺点 |
|------|------|------|
| 前端关联 | 不修改后端，实时性好 | 需要额外请求股票数据 |
| 后端关联 | 数据一致性高 | 需要修改存储结构 |

## 5. 备选方案：后端关联（数据一致性高）

### 5.1 修改data_service.py
```python
def get_bond_ranking(self, limit: int = 30, date: Optional[str] = None,
                    use_mysql: bool = False) -> List[Dict[str, Any]]:
    """获取债券上攻排行（增加股债联动标记）"""
    
    # 1. 获取债券排行
    result = self.get_rising_ranking(asset_type='bond', limit=limit, date=date, use_mysql=use_mysql)
    
    # 2. 获取股票排行中的债券代码
    stock_ranking = self.get_stock_ranking(limit=60, date=date, use_mysql=use_mysql)
    stock_bond_codes = {item['bond_code'] for item in stock_ranking 
                       if item.get('bond_code') and item['bond_code'] != '-'}
    
    # 3. 为债券添加标记
    for item in result:
        item['is_stock_linked'] = item['code'] in stock_bond_codes
        item['code_display'] = f"★{item['code']}" if item['is_stock_linked'] else item['code']
    
    return result
```

## 6. 推荐选择

### 方案对比

| 维度 | 前端关联 | 后端关联 |
|------|---------|---------|
| 实现复杂度 | 低 | 中 |
| 性能影响 | 多1次HTTP请求 | 多1次Redis查询 |
| 数据一致性 | 好（实时） | 好（实时） |
| 维护成本 | 低 | 中 |
| 扩展性 | 好 | 好 |

### 最终推荐：前端关联
- 实现简单，不修改后端存储逻辑
- 实时性好，用户感知延迟低
- 维护成本低
