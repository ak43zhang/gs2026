# 主力净额与累计主力净额计算系统 - 完整文档

## 一、系统概述

### 1.1 功能目标
实时计算股票主力净额（Main Force Net Amount）和累计主力净额（Cumulative Main Force Net），为股票上攻排行提供主力资金流向数据。

### 1.2 核心指标

| 指标 | 说明 | 单位 |
|------|------|------|
| main_net_amount | 主力净额（单周期） | 元 |
| cumulative_main_net | 累计主力净额 | 元 |
| main_behavior | 主力行为类型 | 字符串 |
| main_confidence | 置信度 | 0-1 |

### 1.3 数据来源与存储

**数据来源**:
- 实时行情数据（adata）
- Redis历史数据（上一时刻）

**数据存储**:
- Redis: `monitor_gp_sssj_{date}:{time}`
- MySQL: `monitor_gp_sssj_{date}`

### 2.2 详细计算步骤

#### 步骤1: 获取实时数据

```python
df_now = fetch_all_concurrently(STOCK_CODES)
# 字段: stock_code, short_name, price, volume, amount, change_pct
```

#### 步骤2: 加载历史数据

```python
# 找上一个有数据的时间点
prev_time = redis_util.get_prev_timestamp_with_data(sssj_table, time_full)
df_prev_main = redis_util.load_dataframe_by_time(sssj_table, prev_time)
```

#### 步骤3: 数据预处理

```python
# 统一stock_code格式
df_now['stock_code'] = df_now['stock_code'].astype(str).str.zfill(6)
df_prev_main['stock_code'] = df_prev_main['stock_code'].astype(str).str.zfill(6)

# "-"字符串转为0
if 'cumulative_main_net' in df_prev_main.columns:
    df_prev_main['cumulative_main_net'] = pd.to_numeric(
        df_prev_main['cumulative_main_net'], errors='coerce'
    ).fillna(0)

# 数值字段转换
numeric_cols = ['price', 'volume', 'amount', 'change_pct']
for col in numeric_cols:
    df_now[col] = pd.to_numeric(df_now[col], errors='coerce').fillna(0)
```

#### 步骤4: 计算主力净额

```python
# 4.1 合并数据
merged = pd.merge(
    df_now[['stock_code', 'short_name', 'price', 'volume', 'amount', 'change_pct', 'is_zt']],
    df_prev_main[['stock_code', 'volume', 'amount', 'change_pct']],
    on='stock_code',
    suffixes=('_now', '_prev'),
    how='inner'
)

# 4.2 计算变化量
merged['delta_amount'] = merged['amount_now'] - merged['amount_prev']
merged['delta_volume'] = merged['volume_now'] - merged['volume_prev']
merged['price_change_pct'] = merged['change_pct_now'] - merged['change_pct_prev']

# 4.3 门槛过滤（可选）
mask = (merged['delta_amount'] >= MAIN_FORCE_CONFIG['min_amount']) & \
       (merged['delta_volume'] >= MAIN_FORCE_CONFIG['min_volume'])
valid_data = merged[mask].copy()

# 4.4 计算价格位置
day_high = day_stats.get('day_high', valid_data['price'].max())
day_low = day_stats.get('day_low', valid_data['price'].min())
price_range = day_high - day_low if day_high > day_low else 1.0
valid_data['price_position'] = (valid_data['price'] - day_low) / price_range

# 4.5 计算量能比
avg_volume = valid_data['delta_volume'].median()
valid_data['volume_ratio'] = valid_data['delta_volume'] / avg_volume

# 4.6 判断主力行为
behavior_results = valid_data.apply(
    lambda row: classify_main_force_behavior(
        row['price_position'],
        row['price_change_pct'],
        row['volume_ratio'],
        time_of_day,
        row.get('is_zt', 0) == 1
    ),
    axis=1
)
valid_data['main_behavior'] = behavior_results.apply(lambda x: x['type'])
valid_data['direction'] = behavior_results.apply(lambda x: x['direction'])
valid_data['confidence'] = behavior_results.apply(lambda x: x['confidence'])

# 4.7 计算参与系数
valid_data['participation'] = valid_data['delta_amount'].apply(calculate_participation_ratio)

# 4.8 计算主力净额
valid_data['main_net_amount'] = (
    valid_data['delta_amount'] *
    valid_data['participation'] *
    valid_data['direction'] *
    valid_data['confidence']
).round(2)
```

#### 步骤5: 计算累计主力净额

```python
# 5.1 合并结果到df_now
result_cols = ['stock_code', 'main_net_amount', 'main_behavior', 'confidence']
df_now = df_now.merge(valid_data[result_cols], on='stock_code', how='left')
df_now['main_net_amount'] = df_now['main_net_amount'].fillna(0)
df_now['main_behavior'] = df_now['main_behavior'].fillna('无主力')
df_now['main_confidence'] = df_now['confidence'].fillna(0)

# 5.2 计算累计值
if 'cumulative_main_net' in df_prev_main.columns:
    prev_cumulative = df_prev_main[['stock_code', 'cumulative_main_net']].copy()
    
    df_now = df_now.merge(
        prev_cumulative,
        on='stock_code',
        how='left',
        suffixes=('', '_prev')
    )
    
    df_now['cumulative_main_net_prev'] = df_now['cumulative_main_net_prev'].fillna(0)
    df_now['cumulative_main_net'] = df_now['cumulative_main_net_prev'] + df_now['main_net_amount']
```

## 三、核心算法详解

### 3.1 主力行为分类算法

```python
def classify_main_force_behavior(price_position, price_change_pct, 
                                 volume_ratio, time_of_day, is_zt):
    """
    判断主力行为类型
    
    参数:
        price_position: 价格位置 (0-1)
        price_change_pct: 价格变化率 (%)
        volume_ratio: 成交量比率 (相对于均值)
        time_of_day: 当前时间
        is_zt: 是否涨停
    
    返回:
        {'type': 行为类型, 'direction': 方向系数, 'confidence': 置信度}
    """
```

**行为类型与方向**:

| 行为类型 | 方向 | 置信度 | 触发条件 |
|----------|------|--------|----------|
| 拉高出货 | -1.0 | 0.85 | 高位(≥98%) + 急涨(≥1%) + 极端放量(≥5倍) |
| 真正拉升 | 1.0 | 0.80 | 低位(≤30%) + 放量上涨(≥2倍) |
| 打压吸筹 | 1.0 | 0.80 | 低位(≤30%) + 放量下跌(≥2倍) |
| 恐慌抛售 | -1.0 | 0.75 | 高位(≥90%) + 放量下跌(≥2倍) |
| 早盘缩量涨停 | 1.0 | 0.90 | 涨停 + 早盘 + 缩量(≤0.5倍) |
| 早盘放量涨停 | 1.0 | 0.80 | 涨停 + 早盘 + 放量 |
| 尾盘巨量涨停 | -1.0 | 0.75 | 涨停 + 尾盘 + 巨量(≥3倍) |
| 尾盘涨停 | -0.7 | 0.60 | 涨停 + 尾盘 |
| 盘中涨停 | 1.0 | 0.60 | 涨停 + 盘中 |
| 疑似拉升 | 1.0 | 0.60 | 早盘 + 放量(≥2倍) + 上涨(≥0.3%) |
| 疑似出货 | -1.0 | 0.60 | 尾盘 + 放量(≥2倍) + 上涨(≥0.3%) |
| 不确定 | 0.3/-0.3/0 | 0.30 | 其他情况 |

### 3.2 参与系数算法

```python
def calculate_participation_ratio(delta_amount):
    """
    计算主力参与系数
    
    基于成交额大小判断主力参与程度
    """
    if delta_amount >= 200万:  return 1.0
    elif delta_amount >= 100万: return 0.8 + (delta_amount-100万)/100万 * 0.2
    elif delta_amount >= 50万:  return 0.5 + (delta_amount-50万)/50万 * 0.3
    elif delta_amount >= 30万:  return 0.3 + (delta_amount-30万)/20万 * 0.2
    else:                       return 0.0
```

**参与系数表**:

| 成交额变化 | 参与系数 |
|------------|----------|
| ≥200万 | 1.0 (100%) |
| 100万 | 0.8 (80%) |
| 50万 | 0.5 (50%) |
| 30万 | 0.3 (30%) |
| <30万 | 0 (无主力) |

### 3.3 主力净额计算公式

```
主力净额 = 成交额变化 × 参与系数 × 方向系数 × 置信度

main_net_amount = delta_amount × participation × direction × confidence
```

**示例计算**:

```
场景: 低位放量上涨
- delta_amount = 80万
- participation = 0.6 (80万在50-100万之间)
- direction = 1.0 (买入)
- confidence = 0.8 (真正拉升)

main_net_amount = 80万 × 0.6 × 1.0 × 0.8 = 38.4万
```

### 3.4 累计主力净额计算

```
累计主力净额 = 上一时刻累计值 + 当前主力净额

cumulative_main_net = cumulative_main_net_prev + main_net_amount
```

**计算示例**:

| 时间 | 主力净额 | 上一累计 | 当前累计 |
|------|----------|----------|----------|
| 09:30:00 | 0 | 0 | 0 |
| 09:30:03 | +38.4万 | 0 | 38.4万 |
| 09:30:06 | -10万 | 38.4万 | 28.4万 |
| 09:30:09 | 0 | 28.4万 | 28.4万 |

### 6.1 数据获取


### 6.2 展示格式

```javascript
// 格式化显示
const mainNet = item.main_net_amount || 0;
const mainNetClass = mainNet > 0 ? 'main-net-up' : (mainNet < 0 ? 'main-net-down' : 'main-net-neutral');
```

### 6.3 颜色规则

- 主力净额 > 0: 红色（净流入/买入）
- 主力净额 < 0: 绿色（净流出/卖出）
- 主力净额 = 0: 灰色（无主力）

### 6.4 排序支持

支持按主力净额排序：
- 降序: 净流入大的在前（红色）
- 升序: 净流出大的在前（绿色）
- null值排最后

## 七、注意事项

### 7.1 数据一致性

- 必须保持stock_code格式一致（6位字符串）
- 数值字段必须转换为float类型
- "-"等非数值必须转为0

### 7.2 性能考虑

- 使用向量化操作（避免apply）
- 批量处理而非逐条
- 避免重复计算

### 7.3 边界情况

- 集合竞价期间不计算
- 早盘9:30:00-9:30:15特殊处理
- 数据缺失时使用默认值
