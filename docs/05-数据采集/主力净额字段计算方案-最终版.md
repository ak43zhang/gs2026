在 `monitor_gp_sssj_{date}` 表中增加 `main_net_amount` 字段，用于估算每3秒周期内的主力资金净流入/流出金额。

## 2. 核心计算逻辑

### 2.1 计算公式

```python
主力净额 = 周期成交额 × 主力参与系数 × 方向系数 × 置信度

其中：
- 周期成交额 = 当前累计成交额 - 上一周期累计成交额
- 主力参与系数：0-1，基于成交额大小（30万→0.3，200万→1.0）
- 方向系数：-1到+1，基于价格位置和行为判断
- 置信度：0-1，基于判断的确定性
```

### 2.2 主力参与系数

```python
def calculate_participation_ratio(delta_amount):
    """
    主力参与系数（基于成交额大小）
    """
    if delta_amount >= 2000000:  # 200万
        return 1.0
    elif delta_amount >= 1000000:  # 100万
        return 0.8 + (delta_amount - 1000000) / 1000000 * 0.2
    elif delta_amount >= 500000:  # 50万
        return 0.5 + (delta_amount - 500000) / 500000 * 0.3
    elif delta_amount >= 300000:  # 30万
        return 0.3 + (delta_amount - 300000) / 200000 * 0.2
    else:
        return 0  # 小于30万，认定无主力参与
```

| 成交额 | 参与系数 | 说明 |
|--------|----------|------|
| <30万 | 0 | 散户级，无主力 |
| 30-50万 | 0.3-0.5 | 小单，主力少量参与 |
| 50-100万 | 0.5-0.8 | 中单，主力参与 |
| 100-200万 | 0.8-1.0 | 大单，主力主导 |
| ≥200万 | 1.0 | 超大单，主力完全主导 |

### 2.3 方向系数和置信度

```python
def classify_behavior(price_position, price_change_pct, volume_ratio, time_of_day):
    """
    判断主力行为类型
    """
    
    # 场景1：极高位置 + 急涨 + 极端放量 → 拉高出货
    if price_position >= 0.98 and price_change_pct >= 1.0 and volume_ratio >= 5:
        return {'type': '拉高出货', 'direction': -1, 'confidence': 0.85}
    
    # 场景2：低位 + 放量上涨 → 真正拉升
    if price_position <= 0.3 and price_change_pct >= 0.3 and volume_ratio >= 2:
        return {'type': '真正拉升', 'direction': 1, 'confidence': 0.8}
    
    # 场景3：低位 + 放量下跌 → 打压吸筹
    if price_position <= 0.3 and price_change_pct <= -0.5 and volume_ratio >= 2:
        return {'type': '打压吸筹', 'direction': 1, 'confidence': 0.8}
    
    # 场景4：高位 + 放量下跌 → 恐慌抛售
    if price_position >= 0.9 and price_change_pct <= -0.5 and volume_ratio >= 2:
        return {'type': '恐慌抛售', 'direction': -1, 'confidence': 0.75}
    
    # 场景5：早盘 + 放量上涨 → 疑似拉升
    if time_of_day <= dt_time(10, 0) and volume_ratio >= 2 and price_change_pct >= 0.3:
        return {'type': '疑似拉升', 'direction': 1, 'confidence': 0.6}
    
    # 场景6：尾盘 + 放量上涨 → 疑似出货
    if time_of_day >= dt_time(14, 30) and volume_ratio >= 2 and price_change_pct >= 0.3:
        return {'type': '疑似出货', 'direction': -1, 'confidence': 0.6}
    
    # 场景7：涨停判断（特殊处理）
    if price_change_pct >= 9.5:  # 接近涨停
        return classify_zt_behavior(price_position, time_of_day, volume_ratio)
    
    # 其他场景：不确定
    if price_change_pct >= 0.5:
        return {'type': '不确定', 'direction': 0.3, 'confidence': 0.3}
    elif price_change_pct <= -0.5:
        return {'type': '不确定', 'direction': -0.3, 'confidence': 0.3}
    else:
        return {'type': '不确定', 'direction': 0, 'confidence': 0}
```

## 3. 完整计算流程

### 3.1 主流程

```python
def calculate_main_net_amount(stock_code, current_data, previous_data, 
                             historical_data, day_stats, time_of_day):
    """
    计算主力净额（完整流程）
    """
    
    # 步骤1：计算周期变化
    delta_amount = current_data['amount'] - previous_data['amount']
    delta_volume = current_data['volume'] - previous_data['volume']
    price_change_pct = current_data['change_pct'] - previous_data['change_pct']
    price_now = current_data['price']
    
    # 步骤2：门槛检查
    if delta_amount < 300000 or delta_volume < 20000:
        return {'main_net_amount': 0, 'behavior': '无主力', 'confidence': 0}
    
    # 步骤3：计算价格位置
    day_high = day_stats['day_high']
    day_low = day_stats['day_low']
    price_position = (price_now - day_low) / (day_high - day_low) if day_high > day_low else 0.5
    
    # 步骤4：计算量能比
    avg_volume = historical_data.get('avg_volume', delta_volume)
    volume_ratio = delta_volume / avg_volume if avg_volume > 0 else 1.0
    
    # 步骤5：判断主力行为
    behavior = classify_behavior(price_position, price_change_pct, volume_ratio, time_of_day)
    
    # 步骤6：计算参与系数
    participation = calculate_participation_ratio(delta_amount)
    
    # 步骤7：计算主力净额
    direction = behavior['direction']
    confidence = behavior['confidence']
    
    if direction != 0:
        main_net = delta_amount * participation * direction * confidence
    else:
        main_net = 0
    
    return {
        'main_net_amount': round(main_net, 2),
        'behavior': behavior['type'],
        'direction': direction,
        'confidence': confidence,
        'participation': participation
    }
```

### 3.2 批量计算（向量化）

```python
def batch_calculate_main_net(df_now, df_prev, historical_stats, day_stats, time_of_day):
    """
    批量计算主力净额（向量化优化）
    """
    
    # 合并数据
    merged = pd.merge(
        df_now[['stock_code', 'price', 'volume', 'amount', 'change_pct']],
        df_prev[['stock_code', 'volume', 'amount', 'change_pct']],
        on='stock_code',
        suffixes=('_now', '_prev'),
        how='inner'
    )
    
    # 计算周期变化（向量化）
    merged['delta_amount'] = merged['amount_now'] - merged['amount_prev']
    merged['delta_volume'] = merged['volume_now'] - merged['volume_prev']
    merged['price_change_pct'] = merged['change_pct_now'] - merged['change_pct_prev']
    
    # 门槛过滤（向量化）
    mask = (merged['delta_amount'] >= 300000) & (merged['delta_volume'] >= 20000)
    valid_data = merged[mask].copy()
    
    # 计算指标（向量化）
    valid_data['price_position'] = (valid_data['price'] - day_stats['day_low']) / \
                                   (day_stats['day_high'] - day_stats['day_low'])
    valid_data['volume_ratio'] = valid_data['delta_volume'] / valid_data['avg_volume']
    
    # 分类判断（向量化apply）
    classification = valid_data.apply(
        lambda row: classify_behavior(
            row['price_position'], 
            row['price_change_pct'], 
            row['volume_ratio'],
            time_of_day
        ),
        axis=1
    )
    
    valid_data['direction'] = classification['direction']
    valid_data['confidence'] = classification['confidence']
    valid_data['participation'] = valid_data['delta_amount'].apply(calculate_participation_ratio)
    
    # 计算主力净额（向量化）
    valid_data['main_net_amount'] = (
        valid_data['delta_amount'] * 
        valid_data['participation'] * 
        valid_data['direction'] * 
        valid_data['confidence']
    ).round(2)
    
    return valid_data[['stock_code', 'main_net_amount', 'direction', 'confidence']]
```

## 9. 白话总结

### 9.1 这个方案是做什么的？

```
简单说：
每3秒算一次，每只股票的"主力净额"

主力净额 = 这3秒内，主力资金是流入了还是流出了

正值（+）：主力在买入（钱流入股票）
负值（-）：主力在卖出（钱流出股票）
零（0）：没有主力参与，都是散户在交易
```

### 9.2 怎么判断主力在买入还是卖出？

```
核心逻辑：

1. 先看成交额大小
   - 3秒内成交了多少钱
   - 超过30万才算可能有主力
   - 钱越多，主力参与程度越高

2. 再看价格位置
   - 股价在低位还是高位
   - 低位上涨 → 主力在买
   - 高位上涨 → 主力在卖（拉高出货）
   - 低位下跌 → 主力在买（打压吸筹）
   - 高位下跌 → 主力在卖（恐慌抛售）

3. 再看成交量
   - 比平时大很多（放量）→ 主力在参与
   - 跟平时差不多 → 可能是散户

4. 最后看时间
   - 早盘（9:30-10:00）→ 主力买入概率高
   - 尾盘（14:30-15:00）→ 主力出货概率高
```

### 9.3 几种典型情况

| 情况 | 主力在干嘛 | 主力净额 |
|------|-----------|----------|
| 低位 + 放量 + 上涨 | 主力在买，拉升股价 | 正（流入） |
| 高位 + 急涨 + 天量 | 主力在卖，拉高出货 | 负（流出） |
| 低位 + 放量 + 下跌 | 主力在买，打压吸筹 | 正（流入） |
| 高位 + 放量 + 下跌 | 主力在卖，恐慌抛售 | 负（流出） |
| 早盘涨停 + 缩量 | 主力强烈买入 | 正（大） |
| 尾盘涨停 + 巨量 | 主力在出货 | 负（大） |
| 成交额<30万 | 没有主力，散户在玩 | 0 |

### 9.4 准不准？

```
准确率：约70%

什么意思：
- 10次判断，大概7次是对的
- 3次可能出错

什么时候准：
- 特征明显的（低位放量涨、高位放量跌）→ 准
- 早盘涨停 → 准

什么时候不准：
- 横盘震荡 → 不准
- 尾盘涨停 → 不太准
- 主力很狡猾，故意伪装 → 不准

怎么用：
- 看趋势方向（流入/流出）→ 比较准
- 看具体金额 → 参考，别太较真
- 结合其他指标一起看
```

### 9.5 有什么限制？

```
数据限制：
- 没有逐笔成交明细
- 看不到谁买了谁卖了
- 看不到盘口挂单

所以：
- 只能"估算"，不能"精确"
- 叫"主力净额估算值"更准确
- 不能完全依赖这个指标

建议：
- 作为参考指标之一
- 结合K线、成交量、新闻一起看
- 不要单独用来做交易决策
```
