# 债券排行功能修复文档

## 问题背景

### 现象
- 时间轴点击时债券上攻排行没有涨跌幅和行业数据
- 实时场景下债券没有涨跌幅而股票有

### 影响范围
- 前端页面：`monitor.html` 债券排行显示
- 后端接口：`/api/monitor/attack-ranking/bond`
- 用户无法查看债券的实时涨跌幅和所属行业

---

## 问题分析

### 问题1：at-time 接口遗漏数据增强

#### 根因
`get_ranking_at_time` 接口（用于时间轴点击场景）没有调用 `_enrich_bond_data()` 函数来增强债券数据。

#### 代码位置
```python
# src/gs2026/dashboard2/routes/monitor.py
@monitor_bp.route('/attack-ranking/<asset_type>/at-time', methods=['GET'])
def get_ranking_at_time(asset_type):
    # ... 只返回基础字段，没有涨跌幅和行业信息
```

#### 修复方案
在 `get_ranking_at_time` 函数中添加债券数据增强：

```python
# 【修复】为债券数据添加涨跌幅和行业信息
if asset_type == 'bond' and data and time_str:
    actual_date = date or datetime.now().strftime('%Y%m%d')
    data = _enrich_bond_data(data, actual_date, time_str)
```

---

### 问题2：Config 配置缺失

#### 根因
`dashboard2/config.py` 缺少 `MYSQL_URI` 配置，导致 `_get_bond_change_pct_from_mysql` 函数执行失败。

#### 错误日志
```
MySQL降级查询债券涨跌幅失败: type object 'Config' has no attribute 'MYSQL_URI'
```

#### 修复方案
在 `dashboard2/config.py` 中添加 MySQL 配置：

```python
# MySQL配置（从统一配置读取）
_config = config_util.load_config()
_mysql_config = _config.get('mysql', {})
MYSQL_HOST = _mysql_config.get('host', '192.168.0.101')
MYSQL_PORT = _mysql_config.get('port', 3306)
MYSQL_USER = _mysql_config.get('user', 'root')
MYSQL_PASSWORD = _mysql_config.get('password', '123456')
MYSQL_DATABASE = _mysql_config.get('database', 'gs')

# SQLAlchemy 数据库 URI
MYSQL_URI = f"mysql+pymysql://{MYSQL_USER}:{MYSQL_PASSWORD}@{MYSQL_HOST}:{MYSQL_PORT}/{MYSQL_DATABASE}?charset=utf8"
```

## 相关函数说明

### `_enrich_bond_data(bonds, date, time_str)`
为债券数据添加涨跌幅和行业信息。

**参数：**
- `bonds`: 债券数据列表
- `date`: 日期 YYYYMMDD
- `time_str`: 时间 HH:MM:SS（可选，不传则取最新时间）

**逻辑：**
1. 从 Redis 获取最新时间戳
2. 批量查询涨跌幅（`_get_bond_change_pct_batch`）
3. 批量查询行业（`_get_bond_industry_batch`）
4. 填充数据到债券列表

### `_get_bond_change_pct_batch(date, time_str, bond_codes)`
批量获取债券涨跌幅。

**查询优先级：**
1. 优先从 Redis 查询（`monitor_zq_sssj_{date}:{time_str}`）
2. Redis 未命中，降级到 MySQL 查询

### `_get_bond_industry_batch(bond_codes)`
批量获取债券所属行业。

**实现方式：**
- 通过股债映射缓存反向查找（bond_code → stock_code → industry_name）
