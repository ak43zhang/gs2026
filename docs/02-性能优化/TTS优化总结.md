# TTS语音阅读系统优化总结

## 概述

2026-04-07至2026-04-09期间，对报告中心TTS语音阅读系统进行了全面优化，解决了多个核心问题，提升了阅读体验。

## 问题修复记录

### 1. 第二行无法阅读问题
**问题描述**: 点击播放后，读完第一句直接跳到第三句，跳过第二句

**根本原因**: 
- 前后端MD5哈希算法不一致
- 分段策略不匹配导致段落索引错位

**解决方案**:
- 统一前后端分段策略为`original`（按句分割）
- 修复前端MD5实现，使用更可靠的简化版算法
- 使用文本哈希匹配替代索引匹配

**提交**: `f2c7470`, `1e156e0`

### 2. "隔一个读一个"问题
**问题描述**: 某些段落有音频，某些段落没有，呈现间隔性

**根本原因**: 
- 前后端分段结果不一致
- 按索引匹配导致错位

**解决方案**:
- 实施文本哈希匹配方案
- 每段文本生成MD5哈希作为唯一标识
- 音频文件名使用哈希值

**提交**: `1e156e0`

### 3. 并行加载顺序混乱
**问题描述**: 多段音频同时加载，播放顺序错乱

**解决方案**:
- 实现FIFO队列串行处理
- `_addToQueue()`, `_processQueue()`, `_generateAndPlay()`
- 播放前检查`index === currentSegment`
- `audio.onended`回调确保顺序

**提交**: `20df4ae`

### 4. 跳转弹窗重复
**问题描述**: 点击跳转后，弹窗重复出现

**解决方案**:
- 添加`_isProcessingJump`标志防止重复处理
- 使用`event.preventDefault()`和`stopPropagation()`

**提交**: `943a6d8`

### 5. 上一句/下一句跳过
**问题描述**: 点击导航按钮，跳过多个段落

**根本原因**: `ReportCenter.init()`被调用两次，事件绑定两次

**解决方案**:
- 添加`_initialized`标志防止重复初始化
- 按钮添加`_hasClickHandler`标志防止重复绑定

**提交**: `77a3a94`

### 6. 换行问题修复

#### 6.1 英文句号换行
**问题**: `3.5%`被分割成`3.`和`5%`
**解决**: 从句子结束符列表中移除`.`

#### 6.2 *ST股票换行
**问题**: `*ST美诺`被分割成`*`和`ST美诺`
**解决**: 检测`*ST`前缀，与前一行合并

#### 6.3 序号换行
**问题**: `1.`被当作句子结束
**解决**: 检测序号行（`1.`、`2.`等），作为新句子开始

**提交**: `3dbbc02`, `c6b867a`, `859e074`

## 核心算法

### 文本分割算法
```python
def _split_sentences(text, strategy='original'):
    if strategy == 'original':
        # 按中文标点分割
        sentences = re.split(r'([。！？；\n])', text)
        # 合并标点和内容
        result = []
        for i in range(0, len(sentences)-1, 2):
            sent = sentences[i] + (sentences[i+1] if i+1 < len(sentences) else '')
            if sent.strip():
                result.append(sent.strip())
        return result
```

### 哈希匹配算法
```python
import hashlib

def get_segment_hash(text, strategy):
    """生成段落唯一标识"""
    content = f"{text}_{strategy}"
    return hashlib.md5(content.encode('utf-8')).hexdigest()
```
