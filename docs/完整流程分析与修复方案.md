# 主力净额计算完整流程分析与修复方案

## 一、完整数据链路

### 1. 数据采集层 (monitor_stock.py)

```
实时数据获取 → 主力净额计算 → 存储到Redis/MySQL
```

**关键函数**: `calculate_main_force_and_cumulative()`

**输入**:
- `df_now`: 当前时刻数据（含price, volume, amount, change_pct）
- `df_prev_main`: 上一时刻数据（从Redis加载，含cumulative_main_net）
- `day_stats`: 当日统计数据
- `time_of_day`: 当前时间

**输出**:
- `df_now` 添加字段: `main_net_amount`, `main_behavior`, `main_confidence`, `cumulative_main_net`

### 2. 数据存储层

**Redis Key**: `monitor_gp_sssj_{date}:{time}`
**MySQL表**: `monitor_gp_sssj_{date}`

**存储字段**:
- stock_code, short_name, price, volume, amount, change_pct
- main_net_amount（单条主力净额）
- main_behavior（主力行为类型）
- main_confidence（置信度）
- cumulative_main_net（累计主力净额）

### 3. 数据查询层 (monitor.py)

**关键函数**: `_get_change_pct_and_main_net_batch()`

**查询逻辑**:
1. 优先从Redis获取
2. 回退到MySQL查询
3. 优先使用 `cumulative_main_net`（累计值）
4. 其次使用 `main_net_amount`（单条值）

### 4. 前端展示层 (monitor.html)

**展示逻辑**:
```javascript
const mainNet = item.main_net_amount || 0;
const mainNetClass = mainNet > 0 ? 'main-net-up' : (mainNet < 0 ? 'main-net-down' : 'main-net-neutral');
const mainNetText = mainNet !== 0 ? (mainNet > 0 ? '+' : '') + (mainNet / 10000).toFixed(1) + '万' : '-';
```

**排序支持**: 支持按 `main_net_amount` 排序

## 三、核心逻辑验证

### 主力净额计算逻辑 ✅

```python
# 1. 计算成交额变化
delta_amount = amount_now - amount_prev

# 2. 计算参与系数（基于成交额大小）
participation = calculate_participation_ratio(delta_amount)

# 3. 判断主力行为
direction, confidence = classify_main_force_behavior(...)

# 4. 计算主力净额
main_net_amount = delta_amount * participation * direction * confidence
```

**逻辑正确** ✅

### 累计主力净额计算逻辑 ✅

```python
# 1. 从df_prev_main获取上一时刻累计值
prev_cumulative = df_prev_main[df_prev_main['cumulative_main_net'] != 0]

# 2. 新的累计值 = 上一累计值 + 当前主力净额
cumulative_main_net = cumulative_main_net_prev + main_net_amount
```

**逻辑正确** ✅

### 门槛过滤逻辑 ⚠️

```python
mask = (merged['delta_amount'] >= 300000) & (merged['delta_volume'] >= 20000)
```

**问题**: 门槛30万太高，导致大部分记录被过滤
