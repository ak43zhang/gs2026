# Dashboard2 集成指南

## 项目概述

Dashboard2 是 GS2026 项目的新一代数据采集和监控面板，采用模块化架构设计，支持 31 个采集任务和 5 个监控任务。

## 架构设计

### 4层架构
1. **Presentation Layer**: 前端 UI 组件
2. **Business Layer**: 业务管理器（CollectionManager）
3. **Data Layer**: API 客户端、状态存储
4. **Foundation Layer**: 核心框架（EventBus、Logger 等）

### 组件设计
- **ServiceCard**: 服务卡片组件，显示单个任务
- **ProcessList**: 进程列表组件，显示运行中的进程
- **TabNav**: Tab 导航组件，切换模块

## 启动方式

### 监控类任务
- 直接执行脚本，持续运行
- 使用 `pythonw.exe` 后台运行
- 显示"运行中"状态

### 采集类任务
- 使用包装脚本，批量执行
- 调用指定函数，执行完退出
- 显示"执行中"→"已完成"状态

## API 接口

### 获取模块列表
```
GET /api/collection/modules
```

### 获取模块任务
```
GET /api/collection/<module_id>/tasks
```

### 启动任务
```
POST /api/collection/<module_id>/start/<task_id>
Body: { "start_date": "2026-03-27", "end_date": "2026-03-27" }
```

### 停止任务
```
POST /api/collection/stop/<process_id>
```

### 获取状态
```
GET /api/collection/status
```

### 全部停止
```
POST /api/collection/stopAll
```

## 关键特性

### 1. 无弹窗设计
- 使用 `pythonw.exe` 代替 `python.exe`
- `STARTUPINFO` 隐藏窗口
- 包装脚本重定向输出

### 2. 编辑锁定
- 输入框获得焦点时暂停自动刷新
- 避免参数被重置

### 3. 状态显示
- 运行中（绿色）
- 执行中（黄色）
- 已完成（蓝色）
- 已停止（灰色）

### 4. 前缀匹配停止
- 支持通过 PID 停止进程
- 前缀匹配查找进程

## 已知问题

| 问题 | 描述 | 优先级 |
|------|------|--------|
| 包装脚本换行符 | 某些情况下可能导致语法错误 | 高 |
| 进程状态同步延迟 | 停止后前端状态更新有延迟 | 高 |
| Redis 连接稳定性 | 偶尔出现连接失败 | 高 |
| 前端参数验证 | 缺少参数格式验证 | 中 |
| 错误提示不友好 | API 错误返回不够详细 | 中 |
| 进程日志查看 | 无法在前端查看进程日志 | 中 |
| 任务执行进度 | 采集任务没有进度显示 | 中 |
