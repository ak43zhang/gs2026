# 债券绿名单系统设计文档

## 概述

债券绿名单系统用于筛选符合条件的可转债，作为买入信号参考。系统包含：
1. Spark 批处理程序：生成绿名单数据
2. 缓存系统：Redis 缓存预热
3. 前端展示：债券上攻排行绿名单标记

## 数据流程

```
data_bond_daily_ods (源表)
    ↓
GreenBondList.scala (Spark程序)
    ↓
green_bond_list (MySQL表)
    ↓
green_bond_list_cache.py (缓存管理)
    ↓
Redis Set (缓存)
    ↓
monitor.py (债券排行API)
    ↓
monitor.html (前端展示)
```

## 筛选条件

### 条件1（model=1）
- **规则**: 当前交易日最高涨幅(zgzf) > 4
- **买入日**: 下一个交易日
- **用途**: 捕捉强势突破信号

### 条件2（model=2）
- **规则**: 当前交易日和前一个交易日的实体涨幅(stzf)之和 > 4，且两天均 > 0
- **买入日**: 下一个交易日
- **用途**: 捕捉连续上涨动能

## 时间逻辑

### 关键概念
- **当前交易日**: 数据中的 date 字段
- **下一个交易日**: 根据交易日历 `data_jyrl` 计算的下一个 trade_status=1 的日期
- **买入日(buy_date)**: 下一个交易日

### 示例

假设交易日历：
| trade_date | trade_status |
|------------|--------------|
| 2026-04-09 | 1 |
| 2026-04-10 | 0 (节假日) |
| 2026-04-11 | 0 (周末) |
| 2026-04-12 | 0 (周末) |
| 2026-04-13 | 1 |

债券数据：
| date | bond_code | zgzf | stzf |
|------|-----------|------|------|
| 2026-04-09 | 123001 | 5.5 | 2.5 |

计算过程：
1. 2026-04-09 的 zgzf=5.5 > 4，满足条件1
2. 下一个交易日 = 2026-04-13（跳过节假日和周末）
3. 结果: code=123001, buy_date=2026-04-13, model=1

## 数据库表结构

### green_bond_list
| 字段 | 类型 | 说明 |
|------|------|------|
| code | VARCHAR(10) | 债券代码 |
| buy_date | DATE | 期望买入日期（下一个交易日） |
| model | VARCHAR(10) | 条件编号（1或2） |

### data_jyrl (交易日历)
| 字段 | 类型 | 说明 |
|------|------|------|
| trade_date | DATE | 日期 |
| trade_status | INT | 1=交易日，0=非交易日 |
| day_week | INT | 星期几 |

## 缓存设计

### Redis 键
- `dict:green_bond_list`: DataFrame 格式（兼容旧代码）
- `dict:green_bond_list:set`: Set 格式（O(1)查询）
- `dict:green_bond_list_date`: 当前缓存日期

### 缓存策略
- **预热模式**: SYNC（同步执行）
- **优先级**: CRITICAL（关键）
- **过期时间**: 7天
- **重试次数**: 1次

### 获取时间点排行
```
GET /api/monitor/attack-ranking/bond/at-time?date=YYYYMMDD&time=HH:MM:SS
```

## 注意事项

1. **交易日历**: 必须确保 `data_jyrl` 表数据完整，否则无法正确计算下一个交易日
2. **缓存同步**: 绿名单缓存为同步预热，启动时会阻塞直到完成
3. **日期格式**: 所有日期统一使用 `YYYY-MM-DD` 格式
4. **代码格式**: 债券代码统一补零为6位字符串
