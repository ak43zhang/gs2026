# 主力净额计算逻辑详细分析

## 一、核心计算公式

```
主力净额 = 成交额变化 × 参与系数 × 方向系数 × 置信度

main_net_amount = delta_amount × participation × direction × confidence
```

---

## 二、各参数详解

### 1. 成交额变化 (delta_amount)

**计算**:
```python
delta_amount = amount_now - amount_prev
# 当前时刻成交额 - 上一时刻成交额
```

**含义**: 两个时间点之间的成交额变化量

**示例**:
- 09:30:00 成交额 = 100万
- 09:30:03 成交额 = 150万
- delta_amount = 50万

---

### 2. 参与系数 (participation)

**计算函数**: `calculate_participation_ratio(delta_amount)`

**分级逻辑**:
```python
if delta_amount >= 200万:  participation = 1.0      (100%参与)
elif delta_amount >= 100万: participation = 0.8+   (80%+)
elif delta_amount >= 50万:  participation = 0.5+   (50%+)
elif delta_amount >= 30万:  participation = 0.3+   (30%+)
else:                       participation = 0      (无主力)
```

**含义**: 根据成交额大小判断主力参与程度
- 成交额越大 → 参与系数越高 → 主力净额越大
- 成交额<30万 → 参与系数=0 → 主力净额=0

---

### 3. 方向系数 (direction)

**计算函数**: `classify_main_force_behavior(...)`

**场景判断**:

| 场景 | 条件 | 方向系数 | 含义 |
|------|------|----------|------|
| 拉高出货 | 高位+急涨+放量 | -1.0 | 主力卖出 |
| 真正拉升 | 低位+放量上涨 | 1.0 | 主力买入 |
| 打压吸筹 | 低位+放量下跌 | 1.0 | 主力买入 |
| 恐慌抛售 | 高位+放量下跌 | -1.0 | 主力卖出 |
| 早盘缩量涨停 | 涨停+缩量 | 1.0 | 主力买入 |
| 早盘放量涨停 | 涨停+放量 | 1.0 | 主力买入 |
| 尾盘巨量涨停 | 涨停+巨量 | -1.0 | 主力卖出 |
| 尾盘涨停 | 涨停+正常 | -0.7 | 主力卖出 |
| 疑似拉升 | 早盘+放量涨 | 1.0 | 主力买入 |
| 疑似出货 | 尾盘+放量涨 | -1.0 | 主力卖出 |
| 不确定 | 其他情况 | 0.3/-0.3/0 | 弱信号 |

**方向系数含义**:
- 1.0 = 主力买入（净流入）
- -1.0 = 主力卖出（净流出）
- 0.3 = 弱买入信号
- -0.3 = 弱卖出信号
- 0 = 无明确方向

---

### 4. 置信度 (confidence)

**取值范围**: 0 ~ 0.90

**典型值**:
- 拉高出货: 0.85
- 真正拉升: 0.80
- 打压吸筹: 0.80
- 恐慌抛售: 0.75
- 早盘缩量涨停: 0.90
- 早盘放量涨停: 0.80
- 尾盘巨量涨停: 0.75
- 尾盘涨停: 0.60
- 疑似拉升: 0.60
- 疑似出货: 0.60
- 不确定: 0.30
- 无主力: 0.00

**含义**: 对判断结果的确信程度

---

## 三、主力买入 vs 主力卖出判断

### 主力买入场景 (direction = 1.0)

1. **真正拉升**: 低位 + 放量上涨
   - 价格位置 ≤ 30%
   - 价格变化 ≥ 0.3%
   - 量能比 ≥ 2

2. **打压吸筹**: 低位 + 放量下跌
   - 价格位置 ≤ 30%
   - 价格变化 ≤ -0.5%
   - 量能比 ≥ 2

3. **涨停买入**: 早盘涨停
   - 涨停状态
   - 早盘时段 (9:30-10:00)

4. **疑似拉升**: 早盘放量上涨
   - 早盘时段
   - 量能比 ≥ 2
   - 价格变化 ≥ 0.3%

### 主力卖出场景 (direction = -1.0)

1. **拉高出货**: 高位 + 急涨 + 极端放量
   - 价格位置 ≥ 98%
   - 价格变化 ≥ 1.0%
   - 量能比 ≥ 5

2. **恐慌抛售**: 高位 + 放量下跌
   - 价格位置 ≥ 90%
   - 价格变化 ≤ -0.5%
   - 量能比 ≥ 2

3. **尾盘出货**: 尾盘巨量涨停
   - 涨停状态
   - 尾盘时段 (14:30-15:00)
   - 量能比 ≥ 3

4. **疑似出货**: 尾盘放量上涨
   - 尾盘时段
   - 量能比 ≥ 2
   - 价格变化 ≥ 0.3%

## 四、主力净额正负含义

### 主力净额 > 0 (净流入/买入)

**计算结果**:
```
main_net_amount = delta_amount × participation × 1.0 × confidence
                = 正值
```

**场景**:
- 真正拉升
- 打压吸筹
- 早盘涨停
- 疑似拉升

**前端展示**: 红色，表示主力买入

---

### 主力净额 < 0 (净流出/卖出)

**计算结果**:
```
main_net_amount = delta_amount × participation × (-1.0) × confidence
                = 负值
```

**场景**:
- 拉高出货
- 恐慌抛售
- 尾盘巨量涨停
- 疑似出货

**前端展示**: 绿色，表示主力卖出

---

### 主力净额 = 0

**原因**:
1. 成交额变化 < 30万（参与系数=0）
2. 被门槛过滤（delta_amount < 30万 或 delta_volume < 200手）
3. 方向系数=0（不确定场景）
4. 置信度=0（无主力）

**前端展示**: "-"，表示无主力或数据缺失

---

## 五、累计主力净额计算

### 计算公式

```
当前累计值 = 上一时刻累计值 + 当前主力净额

cumulative_main_net = cumulative_main_net_prev + main_net_amount
```

### 计算示例

| 时间 | 主力净额 | 上一累计 | 当前累计 | 说明 |
|------|----------|----------|----------|------|
| 09:30:00 | 0 | 0 | 0 | 初始值 |
| 09:30:03 | 0 | 0 | 0+0=0 | 无主力 |
| 09:30:06 | 0 | 0 | 0+0=0 | 无主力 |
| 09:30:09 | +50万 | 0 | 0+50万=50万 | 主力买入 |
| 09:30:12 | 0 | 50万 | 50万+0=50万 | 保持 |
| 09:30:15 | -20万 | 50万 | 50万-20万=30万 | 主力卖出 |
| 09:30:18 | 0 | 30万 | 30万+0=30万 | 保持 |

## 六、逻辑正确性评估

### 正确的地方 ✅

1. **方向判断逻辑合理**
   - 低位放量上涨 = 买入
   - 高位放量下跌 = 卖出
   - 符合技术分析常识

2. **参与系数分级合理**
   - 成交额越大，主力参与程度越高
   - 小成交额自然衰减到0

3. **置信度机制合理**
   - 明确场景（如涨停）置信度高
   - 模糊场景置信度低

4. **累计值计算正确**
   - 逐时间累加
   - 保持历史值

### 可能的问题 ⚠️

1. **门槛过滤过严**
   - 30万门槛导致90%股票被过滤
   - 建议降低到1万

2. **inner join丢失股票**
   - 部分股票在df_prev_main中缺失
   - 建议改为left join

3. **尾盘涨停判断为卖出**
   - 尾盘涨停 direction=-0.7
   - 这是设计意图（防止追高），但可能误判

---

## 七、结论

### 主力净额计算逻辑 ✅ 正确

- 买入/卖出判断基于价格位置、成交量、时间
- 符合技术分析原理
- 参数设置合理

### 建议优化

1. **降低门槛**: 30万 → 1万，让更多股票参与
2. **left join**: 保留所有股票，避免丢失
3. **尾盘涨停**: 可考虑调整为中性或弱买入
