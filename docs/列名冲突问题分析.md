# 主力净额计算 - 列名冲突问题分析与修复

## 错误信息
```
计算主力净额失败: 'main_net_amount'
```

## 问题分析

### 代码流程

**第449-452行：初始化字段**
```python
# 初始化字段
df_now['main_net_amount'] = 0.0
df_now['main_behavior'] = '无主力'
df_now['main_confidence'] = 0.0
df_now['cumulative_main_net'] = 0.0
```

**第556-557行：合并结果**
```python
result_cols = ['stock_code', 'main_net_amount', 'main_behavior', 'confidence']
df_now = df_now.merge(valid_data[result_cols], on='stock_code', how='left')
```

### 问题根因

`df_now` 和 `valid_data` 都有 `main_net_amount` 等列，merge时产生冲突：

```
df_now columns: [..., 'main_net_amount', 'main_behavior', 'main_confidence', ...]
valid_data columns: ['stock_code', 'main_net_amount', 'main_behavior', 'confidence']

merge后列名变成：
- main_net_amount_x (来自df_now)
- main_net_amount_y (来自valid_data)
- main_behavior_x
- main_behavior_y
- confidence
```

**第558行报错**：
```python
df_now['main_net_amount'] = df_now['main_net_amount'].fillna(0)
# 报错！因为列名是 main_net_amount_x 或 main_net_amount_y，不是 main_net_amount
```

## 修复方案

### 方案A：先删除df_now中的旧列（推荐）

在merge前删除 `df_now` 中的冲突列：

```python
# 修复前
result_cols = ['stock_code', 'main_net_amount', 'main_behavior', 'confidence']
df_now = df_now.merge(valid_data[result_cols], on='stock_code', how='left')

# 修复后
# 先删除df_now中的冲突列，避免merge时产生_x/_y后缀
cols_to_drop = ['main_net_amount', 'main_behavior', 'confidence']
for col in cols_to_drop:
    if col in df_now.columns:
        df_now = df_now.drop(columns=[col])

result_cols = ['stock_code', 'main_net_amount', 'main_behavior', 'confidence']
df_now = df_now.merge(valid_data[result_cols], on='stock_code', how='left')
```

### 方案B：使用suffixes处理冲突

```python
# 修复后
df_now = df_now.merge(
    valid_data[result_cols], 
    on='stock_code', 
    how='left',
    suffixes=('', '_new')  # valid_data的列加_new后缀
)

# 然后用新列覆盖旧列
if 'main_net_amount_new' in df_now.columns:
    df_now['main_net_amount'] = df_now['main_net_amount_new'].fillna(df_now['main_net_amount'])
    df_now = df_now.drop(columns=['main_net_amount_new'])
```

### 方案C：不初始化字段

```python
# 修复前（第449-452行）
df_now['main_net_amount'] = 0.0
df_now['main_behavior'] = '无主力'
df_now['main_confidence'] = 0.0
df_now['cumulative_main_net'] = 0.0

# 修复后：删除初始化，让merge直接添加列
# 然后在fillna时处理缺失值
```

## 推荐方案

**方案A** - 先删除冲突列

理由：
1. 逻辑清晰：先清理旧数据，再合并新数据
2. 代码简洁，易于理解
3. 避免suffixes带来的复杂处理
