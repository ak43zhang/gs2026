# 涨停查询性能分析

## 当前实现分析

### 查询流程

```
1. 从宽表缓存获取所有有选中标签的股票
   - 遍历所有股票（约5000只）
   - 检查是否匹配标签
   - 假设匹配1000只
   
2. 查询这1000只股票在指定时间点的价格
   a. 先尝试Redis
      - 获取整个时间点的所有股票数据（可能5000+条记录）
      - 遍历筛选出匹配的1000只
   b. Redis失败则查询MySQL
      - SQL: SELECT ... WHERE stock_code IN ('000001','000002',...1000个)
      - 大IN列表可能导致性能问题
      
3. 查询转债价格
   - 同样的问题：大IN列表
   
4. 组装结果
   - 遍历1000只股票
```

### 性能瓶颈

**瓶颈1：Redis查询**
```python
# 获取整个时间点的所有数据
df = pd.DataFrame(json.loads(data))  # 可能5000+条
for _, row in df.iterrows():         # 遍历5000+
    if row['stock_code'] in stock_codes:  # 1000次匹配
        result[row['stock_code']] = {...}
```
问题：即使只需要1000只，也要加载和遍历5000+条

**瓶颈2：MySQL大IN列表**
```python
placeholders = ','.join([f"'{code}'" for code in stock_codes])  # 1000个
sql = f"... WHERE stock_code IN ({placeholders})"
```
问题：IN列表太长，MySQL优化器可能选择全表扫描

**瓶颈3：多次Redis连接**
```python
# 股票价格查询
redis_client = _get_redis_client()  # 第1次

# 转债价格查询  
redis_client = _get_redis_client()  # 第2次
```

## 优化方案

### 方案A：批量查询优化（推荐）

**优化1：Redis使用批量获取**
```python
# 改为批量获取指定股票
for code in stock_codes:
    redis_key = f"monitor_gp_sssj_{date_str}:{time}:{code}"
    data = redis_client.get(redis_key)
    if data:
        result[code] = json.loads(data)
```
但当前Redis数据结构是整个DataFrame，不是单只股票。

**优化2：MySQL分批查询**
```python
# 将1000只股票分批，每批100只
batch_size = 100
for i in range(0, len(stock_codes), batch_size):
    batch = stock_codes[i:i+batch_size]
    placeholders = ','.join([f"'{code}'" for code in batch])
    sql = f"... WHERE stock_code IN ({placeholders})"
    # 查询并合并结果
```

**优化3：使用JOIN替代大IN**
```python
# 创建临时表或子查询
sql = f"""
    SELECT s.stock_code, s.short_name, s.price, s.change_pct
    FROM {table_name} s
    INNER JOIN (
        SELECT '{stock_codes[0]}' as code
        UNION ALL SELECT '{stock_codes[1]}'
        ...
    ) codes ON s.stock_code = codes.code
    WHERE s.time = '{time}'
"""
```

### 方案B：预聚合优化

**思路：** 在数据写入时，同时写入按标签聚合的数据

```python
# 写入Redis时，同时写入标签索引
redis_client.set(f"tag_index:{date_str}:{time}:半导体", json.dumps(['000001','000002',...]))
redis_client.set(f"tag_index:{date_str}:{time}:芯片", json.dumps(['000001','000003',...]))

# 查询时直接获取
stock_codes = json.loads(redis_client.get(f"tag_index:{date_str}:{time}:半导体"))
```

优点：查询时无需遍历所有股票
缺点：需要修改数据写入逻辑，增加存储

### 方案C：异步并行查询

**思路：** 股票价格查询和转债价格查询并行执行

```python
import asyncio

async def query_prices_async(stock_codes, date_str, time):
    # 股票价格查询
    price_task = asyncio.create_task(_query_prices_at_time(stock_codes, date_str, time))
    
    # 转债价格查询
    bond_codes = [...]
    bond_task = asyncio.create_task(_get_bond_prices_at_time(date_str, time, bond_codes))
    
    # 等待两个任务完成
    price_data, bond_prices = await asyncio.gather(price_task, bond_task)
    return price_data, bond_prices
```

### 方案D：缓存优化

**思路：** 缓存查询结果

```python
# 使用Redis缓存查询结果
cache_key = f"ztb_filter:{date_str}:{time}:{hash(str(selected_tags))}"
cached = redis_client.get(cache_key)
if cached:
    return json.loads(cached)

# 查询并缓存
result = {...}
redis_client.setex(cache_key, 60, json.dumps(result))  # 缓存60秒
```

## 推荐方案

**方案A-2 + 方案C：分批查询 + 异步并行**

1. **股票价格查询分批**（每批100只）
2. **转债价格查询分批**（每批100只）
3. **两个查询并行执行**

预期性能提升：
- 当前：串行查询，大IN列表
- 优化：并行查询，小IN列表
- 提升：2-3倍
