## 一、设计目标

开发一个通用的离线数据验证工具，支持多种分析模块的数据质量检测，具备可扩展的验证规则引擎。

## 三、命令行接口

### 3.1 参数设计

```bash
python data_validator.py \
    --start-date 20260401 \      # 开始日期 (YYYYMMDD)
    --end-date 20260422 \        # 结束日期 (YYYYMMDD)
    --types ztb,news \           # 验证类型，逗号分隔
    --mode check \               # 执行模式: check/fix/report/interactive
    --output ./reports \         # 报告输出目录
    --config ./validation_config.json  # 验证规则配置文件
```

### 3.2 参数说明

| 参数 | 简写 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `--start-date` | `-s` | 是 | - | 开始日期，格式YYYYMMDD |
| `--end-date` | `-e` | 是 | - | 结束日期，格式YYYYMMDD |
| `--types` | `-t` | 否 | all | 验证类型: ztb/news/notice/domain/all |
| `--mode` | `-m` | 否 | check | 执行模式 |
| `--output` | `-o` | 否 | ./reports | 报告输出目录 |
| `--config` | `-c` | 否 | 内置配置 | 验证规则配置文件 |

### 3.3 执行模式

| 模式 | 说明 |
|------|------|
| `check` | 仅检测，输出异常数据列表 |
| `fix` | 检测并自动清理异常数据 |
| `report` | 仅生成报告，不输出到控制台 |
| `interactive` | 交互模式，逐条确认清理 |

### 3.4 使用示例

```bash
# 检测2026-04-22的涨停分析和新闻分析数据
python data_validator.py -s 20260422 -e 20260422 -t ztb,news

# 检测并自动清理
python data_validator.py -s 20260422 -e 20260422 -t news -m fix

# 检测2026-04-01到2026-04-22的所有数据
python data_validator.py -s 20260401 -e 20260422 -t all -m check

# 交互模式
python data_validator.py -s 20260422 -e 20260422 -t news -m interactive

# 使用自定义规则配置
python data_validator.py -s 20260422 -e 20260422 -t news -c ./my_rules.json
```

class RuleEngine:
    """规则引擎"""
    
    def __init__(self, rules: List[ValidationRule]):
        self.rules = rules
    
    def validate(self, record: Dict[str, Any]) -> List[str]:
        """验证单条记录，返回错误列表"""
        errors = []
        for rule in self.rules:
            error = self._check_rule(rule, record)
            if error:
                errors.append(error)
        return errors
    
    def _check_rule(self, rule: ValidationRule, record: Dict[str, Any]) -> Optional[str]:
        """检查单条规则"""
        value = record.get(rule.field)
        
        if rule.type == 'range':
            min_val = rule.params.get('min')
            max_val = rule.params.get('max')
            if not (min_val <= value <= max_val):
                return f"{rule.name}: {value} 超出区间 [{min_val}, {max_val}]"
        
        elif rule.type == 'enum':
            allowed = rule.params.get('values', [])
            if value not in allowed:
                return f"{rule.name}: '{value}' 不在允许值 {allowed} 中"
        
        elif rule.type == 'formula':
            formula = rule.params.get('formula')
            tolerance = rule.params.get('tolerance', 0)
            expected = self._eval_formula(formula, record)
            if abs(value - expected) > tolerance:
                return f"{rule.name}: {value} 与公式计算值 {expected} 不一致"
        
        elif rule.type == 'not_null':
            if value is None or value == '':
                return f"{rule.name}: 字段不能为空"
        
        elif rule.type == 'regex':
            import re
            pattern = rule.params.get('pattern', '')
            if not re.match(pattern, str(value)):

                name=rule_config['name'],
                field=rule_config['field'],
                type=rule_config['type'],
                params={k: v for k, v in rule_config.items() 
                       if k not in ['name', 'field', 'type', 'description']},
                description=rule_config.get('description', '')
            ))
        return rules

                    errors=errors,
                    raw_data=record
