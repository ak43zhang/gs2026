# 新闻分析热点板块逻辑变更记录

**变更日期**: 2026-04-17  
**变更类型**: 功能优化  
**影响范围**: 新闻分析 - 热点板块排行  
**相关文件**: `src/gs2026/dashboard2/services/news_service.py`

---

## 一、变更概述

### 变更前
热点板块排行只统计 **"重大利好"** 新闻（`news_type='利好' AND news_size='重大'`）

### 变更后
热点板块排行统计 **所有"利好"新闻**（`news_type='利好'`，不限制 `news_size`）

---

## 二、变更原因

用户反馈当前热点板块统计过于严格，只展示"重大"利好消息导致：
1. 热点板块数量过少，无法反映真实的板块热度
2. "大"、"中"、"小"利好消息对板块热度同样有参考价值
3. 与用户的业务预期不符

---

## 三、代码变更详情

### 文件位置
`src/gs2026/dashboard2/services/news_service.py`

### 函数
`get_hot_sectors(date: str = None, top_n: int = 10)`

### SQL 变更

```sql
-- 变更前
SELECT sector_name, COUNT(*) as cnt, ROUND(AVG(composite_score), 1) as avg_score
FROM (
    SELECT JSON_UNQUOTE(jt.sector_name) as sector_name, composite_score
    FROM analysis_news_detail_2026,
         JSON_TABLE(sectors, '$[*]' COLUMNS (sector_name VARCHAR(100) PATH '$')) AS jt
    WHERE publish_time BETWEEN '{start_time}' AND '{end_time}'
      AND news_type = '利好'
      AND news_size = '重大'  -- 限制只统计重大利好
) AS t
GROUP BY sector_name
ORDER BY cnt DESC
LIMIT {top_n}

-- 变更后
SELECT sector_name, COUNT(*) as cnt, ROUND(AVG(composite_score), 1) as avg_score
FROM (
    SELECT JSON_UNQUOTE(jt.sector_name) as sector_name, composite_score
    FROM analysis_news_detail_2026,
         JSON_TABLE(sectors, '$[*]' COLUMNS (sector_name VARCHAR(100) PATH '$')) AS jt
    WHERE publish_time BETWEEN '{start_time}' AND '{end_time}'
      AND news_type = '利好'  -- 只限制利好，不限制大小
) AS t
GROUP BY sector_name
ORDER BY cnt DESC
LIMIT {top_n}
```

### 关键改动
- **删除**: `AND news_size = '重大'` 条件
- **影响**: 统计范围扩大，包含重大/大/中/小所有级别的利好消息

---

## 四、影响分析

### 数据影响
| 指标 | 变更前 | 变更后 | 预期变化 |
|------|--------|--------|----------|
| 统计范围 | 仅重大利好 | 所有利好 | 数据量增加 |
| 热点板块数量 | 较少 | 较多 | 更全面的热度反映 |
| 板块排名 | 基于重大利好 | 基于全部利好 | 排名可能变化 |

### 业务影响
- ✅ 更准确地反映板块整体热度
- ✅ 更多板块有机会进入热点排行
- ⚠️ 部分原本靠"重大"利好上榜的板块排名可能下降
