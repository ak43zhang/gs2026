## 二、设计方案

### 2.1 数据库表结构修改

```sql
-- 为 analysis_news_detail_2026 表添加 deep_analysis 字段（JSON 类型）
ALTER TABLE analysis_news_detail_2026 
ADD COLUMN deep_analysis JSON NULL COMMENT '深度分析（JSON数组）'
AFTER sector_details;
```

**字段说明**:
- 名称: `deep_analysis`
- 类型: `JSON`
- 可空: `YES`
- 默认值: `NULL`
- 存储格式: JSON 数组（MySQL 原生 JSON 类型）
- 示例: `["成本控制: 显著降低成本，因为... (5分)", "运营效率: 大幅提升... (5分)"]`

**JSON 类型优势**:
- 自动 JSON 格式验证
- 支持 JSON 函数查询（`JSON_EXTRACT`, `JSON_CONTAINS` 等）
- 存储空间比 TEXT 更紧凑
- 支持部分更新（MySQL 8.0+）

## 四、兼容性处理

### 历史数据
- 历史数据的 `deep_analysis` 字段为 `NULL`
- 前端展示时判断：如果为空数组或 NULL，隐藏深度分析区块

### 回滚方案
```sql
-- 如需回滚，删除字段
ALTER TABLE analysis_news_detail_2026 DROP COLUMN deep_analysis;
```

### JSON 类型注意事项
- MySQL 5.7.8+ 或 MySQL 8.0+ 支持 JSON 类型
- 插入时会自动验证 JSON 格式，非法 JSON 会报错
- 查询时返回字符串，需要应用层 `json.loads()` 解析
