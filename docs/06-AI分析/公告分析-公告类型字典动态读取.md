# 公告分析 - 公告类型字典动态读取与notice_category字段

## 修改日期
2026-04-30

## 修改背景

公告分析Prompt中的"公告类型字典"原来是**硬编码**在代码中的272个类型，维护困难（新增/修改类型需要改代码重新部署）。同时，AI返回的`公告类型`字段没有被提取和存储到数据库中，造成信息丢失。

## 修改内容

### 1. 数据库表结构

```sql
ALTER TABLE analysis_notice_detail_2026 
ADD COLUMN notice_category VARCHAR(64) DEFAULT '' AFTER notice_type;
```

**说明：**
- `notice_type` 字段（已有）= `enum('利好','利空','中性')` — 存储消息类型
- `notice_category` 字段（新增）= `VARCHAR(64)` — 存储公告类型分类（如"业绩暴增超预期"、"定向增发"等）

### 2. 字典动态读取

**字典来源表：** `notice_type`（272条记录，`flag=1`启用）

```python
# get_notice_analysis()中新增：
notice_type_dic_sql = "select type from notice_type where flag='1'"
notice_type_dic_str = ','.join(pd.read_sql(notice_type_dic_sql, conn)['type'].astype(str))
```

**Prompt中使用：**
```python
# 修改前（硬编码272个类型）
公告类型（公告类型字典：业绩暴增超预期,业绩预增合预期,...年报盈转亏）

# 修改后（动态读取）
公告类型（公告类型字典：""" + notice_type_dic_str + """）
```

### 3. 提取和存储

**`_extract_notice_record()`新增：**
```python
'notice_category': str(notice.get('公告类型', '')).strip()[:64],
```

**`process_notice()`的key_fields新增：**
```python
key_fields = ['risk_level', 'notice_type', 'notice_category',  # ← 新增
              'judgment_basis', 'key_points', ...]
```

## 数据流

```
get_notice_analysis()
    ├─ 查询 notice_type 表 → notice_type_dic_str（272个类型逗号分隔）
    └─ deepseek_ai(query_list, notice_type_dic_str, ...)
        ├─ Prompt: 公告类型（公告类型字典：{notice_type_dic_str}）
        ├─ AI返回JSON: {"公告集合": [{"公告类型": "业绩暴增超预期", ...}]}
        └─ process_notice(json_data)
            ├─ _extract_notice_record() → notice_category = "业绩暴增超预期"
            └─ batch_insert_on_duplicate() → 写入 notice_category 字段
```

## 修改原因

1. **可维护性：** 新增/修改公告类型只需更新`notice_type`表，无需改代码重新部署
2. **数据完整性：** AI已经返回了`公告类型`字段，之前没有存储造成信息丢失
3. **一致性：** 与板块字典、概念字典的模式统一

## 注意事项

1. **ALTER TABLE需手动执行** — 由于表锁，无法自动执行，需在数据库空闲时运行
2. **向后兼容** — `notice_category`字段默认为空字符串，旧数据不受影响
3. **字典维护** — 后续新增公告类型只需在`notice_type`表INSERT新记录并设`flag=1`
