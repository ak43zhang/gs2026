# 主力净额计算问题分析

## 问题现象

当前计算结果全部为：
- `main_net_amount = 0`
- `main_behavior = '无主力'`
- `main_confidence = 0`

## 根因分析

### 1. 门槛过滤太严格

```python
MAIN_FORCE_CONFIG = {
    'min_amount': 300000,      # 30万 - 太高！
    'min_volume': 20000,       # 200手
}

# 过滤代码
mask = (merged['delta_amount'] >= 300000) & \
       (merged['delta_volume'] >= 20000)
```

### 2. 问题详解

`delta_amount` 是主力净额（成交额变化 * 0.3），不是成交额本身。

**实际情况**：
- 大部分股票的成交额变化在几万到几十万之间
- 乘以0.3后，主力净额只有几万
- 远小于30万门槛，全部被过滤

### 3. 数据验证

以600418为例：
```
09:30:03: amount=39,890,000, delta=1,990,000, main_net=597,000
09:30:09: amount=44,770,000, delta=4,880,000, main_net=1,464,000
```

- 成交额变化：199万、488万
- 主力净额：59.7万、146.4万
- 但门槛是30万，只有部分记录通过

## 解决方案

### 方案A：降低门槛（推荐）

```python
MAIN_FORCE_CONFIG = {
    'min_amount': 10000,      # 1万 - 降低门槛
    'min_volume': 1000,       # 10手
}
```

### 方案B：使用成交额变化而非主力净额

修改过滤条件，使用原始成交额变化：
```python
# 使用原始成交额变化过滤
raw_delta = merged['amount_now'] - merged['amount_prev']
mask = (raw_delta >= 300000) & (merged['delta_volume'] >= 20000)
```

### 方案C：取消门槛过滤

```python
# 直接计算所有数据
valid_data = merged.copy()
```

## 建议

**推荐方案A**：降低门槛到1万

理由：
1. 保留过滤机制，排除极小成交
2. 让更多股票参与计算
3. 主力净额通过参与系数自然衰减
