# 主力净额计算逻辑修复方案 v3

## 核心优化

**关键改进：累计主力净额复用主力净额的 `df_prev_main`**

```
原方案:
1. 主力净额: 找上一个有数据点 → 计算 main_net_amount
2. 累计净额: 再找上一个有累计值点 → 查询 MySQL → 计算 cumulative_main_net

新方案:
1. 主力净额: 找上一个有数据点 → 计算 main_net_amount
2. 累计净额: 直接使用 df_prev_main 中的 cumulative_main_net → 计算新的累计值

优势: 减少一次查询，直接使用已加载的数据
```

```python
def get_prev_timestamp_with_data(table_name: str, current_time: str, redis_client=None) -> str:
    """
    获取上一个有数据的时间点
    
    优先Redis时间戳列表，其次MySQL查询
    """
    prev_time = None
    
    # 方法1: Redis时间戳列表
    if redis_client:
        try:
            ts_key = f"{table_name}:timestamps"
            all_ts = redis_client.lrange(ts_key, 0, -1)
            
            if all_ts:
                timestamps = sorted([
                    t.decode('utf-8') if isinstance(t, bytes) else t
                    for t in all_ts
                ])
                
                for ts in reversed(timestamps):
                    if ts < current_time:
                        prev_time = ts
                        break
        except Exception as e:
            logger.warning(f"从Redis获取时间戳失败: {e}")
    
    # 方法2: MySQL查询
    if not prev_time:
        try:
            query = f"""
                SELECT MAX(time) as prev_time
                FROM {table_name}
                WHERE time < '{current_time}'
                LIMIT 1
            """
            result = pd.read_sql(query, con=engine)
            if not result.empty and result.iloc[0]['prev_time']:
                prev_time = result.iloc[0]['prev_time']
        except Exception as e:
            logger.warning(f"从MySQL获取上一时间失败: {e}")
    
    return prev_time


def load_dataframe_by_time(table_name: str, time_str: str) -> pd.DataFrame:
    """
    根据时间点加载DataFrame
    优先Redis，其次MySQL
    """
    # Redis
    redis_key = f"{table_name}:{time_str}"
    df = redis_util.load_dataframe_by_key(redis_key, use_compression=False)
    
    if df is not None and not df.empty:
        return df
    
    # MySQL回退
    try:
        query = f"SELECT * FROM {table_name} WHERE time = '{time_str}'"
        df = pd.read_sql(query, con=engine)
        return df
    except Exception as e:
        logger.error(f"从MySQL加载数据失败: {e}")
        return pd.DataFrame()
```

### 修改点2：新增函数 - 计算主力净额和累计值（一体化）

**文件**: `monitor_stock.py`

```python
def calculate_main_force_with_cumulative(df_now: pd.DataFrame,
                                         df_prev: pd.DataFrame,
                                         table_name: str,
                                         current_time: str,
                                         day_stats: dict,
                                         time_of_day: dt_time) -> pd.DataFrame:
    """
    计算主力净额和累计主力净额（一体化）
    
    Args:
        df_now: 当前时刻数据
        df_prev: 上一时刻数据（已加载，包含cumulative_main_net）
        table_name: 表名
        current_time: 当前时间
        day_stats: 当日统计
        time_of_day: 当前时间对象
    
    Returns:
        添加了 main_net_amount, main_behavior, main_confidence, cumulative_main_net 的 DataFrame
    """
    # 1. 计算主力净额
    main_force_result = calculate_main_force_net_amount(
        df_now, df_prev, day_stats, time_of_day
    )
    
    if main_force_result.empty:
        # 无结果，添加空字段
        df_now['main_net_amount'] = 0.0
        df_now['main_behavior'] = '无主力'
        df_now['main_confidence'] = 0.0
        df_now['cumulative_main_net'] = 0.0
        return df_now
    
    # 2. 合并主力净额数据
    df_now = df_now.merge(
        main_force_result[['stock_code', 'main_net_amount', 'main_behavior', 'main_confidence']],
        on='stock_code',
        how='left'
    )
    df_now['main_net_amount'] = df_now['main_net_amount'].fillna(0)
    df_now['main_behavior'] = df_now['main_behavior'].fillna('无主力')
    df_now['main_confidence'] = df_now['main_confidence'].fillna(0)
    
    # 3. 【关键】计算累计主力净额 - 直接使用 df_prev 中的 cumulative_main_net
    df_now = calculate_cumulative_main_net_v3(df_now, df_prev)
    
    return df_now


def calculate_cumulative_main_net_v3(df_now: pd.DataFrame, df_prev: pd.DataFrame) -> pd.DataFrame:
    """
    计算累计主力净额（v3 - 直接使用df_prev中的累计值）
    
    不复查MySQL，直接使用已加载的df_prev中的cumulative_main_net
    
    Args:
        df_now: 当前时刻数据（包含main_net_amount）
        df_prev: 上一时刻数据（包含cumulative_main_net）
    
    Returns:
        添加了cumulative_main_net的DataFrame
    """
    # 初始化累计值为当前值
    df_now['cumulative_main_net'] = df_now['main_net_amount'].fillna(0)
    
    if df_prev is None or df_prev.empty:
        # 无上一时刻数据，使用当前值
        return df_now
    
    try:
        # 检查df_prev是否有cumulative_main_net字段
        if 'cumulative_main_net' not in df_prev.columns:
            # 无累计字段，使用当前值
            return df_now
        
        # 【关键】直接使用df_prev中的cumulative_main_net
        # 筛选有累计值的数据
        prev_cumulative = df_prev[df_prev['cumulative_main_net'] != 0][['stock_code', 'cumulative_main_net']].copy()
        
        if prev_cumulative.empty:
            # 上一时刻无累计值，使用当前值
            return df_now
        
        # 类型统一
        df_now['stock_code'] = df_now['stock_code'].astype(str)
        prev_cumulative['stock_code'] = prev_cumulative['stock_code'].astype(str)
        
        # 合并上一时刻的累计值
        df_now = df_now.merge(
            prev_cumulative,
            on='stock_code',
            how='left',
            suffixes=('', '_prev')
        )
        
        # 计算新的累计值 = 上一累计值 + 当前值
        df_now['cumulative_main_net_prev'] = df_now['cumulative_main_net_prev'].fillna(0)
        df_now['cumulative_main_net'] = df_now['cumulative_main_net_prev'] + df_now['main_net_amount'].fillna(0)
        
        # 删除临时列
        df_now = df_now.drop(columns=['cumulative_main_net_prev'], errors='ignore')
        
        non_zero_count = (df_now['cumulative_main_net'] != 0).sum()
        logger.info(f"累计主力净额计算完成: {non_zero_count}只股票，基于上一时刻数据")
        
    except Exception as e:
        logger.error(f"计算累计主力净额失败: {e}")
        # 出错时使用当前值
        df_now['cumulative_main_net'] = df_now['main_net_amount'].fillna(0)
    
    return df_now
```

## 计算流程对比

### v2方案（原）
```
1. 找上一个有数据点 → 加载df_prev_main
2. 计算main_net_amount
3. 再找上一个有累计值点 → 查询MySQL
4. 计算cumulative_main_net

问题: 第3步重复查询，且可能和df_prev_main不是同一时间
```

### v3方案（新）
```
1. 找上一个有数据点 → 加载df_prev_main
2. 计算main_net_amount
3. 【直接使用】df_prev_main中的cumulative_main_net
4. 计算cumulative_main_net = prev_cum + current

优势: 无重复查询，直接使用已加载数据
```

## 时间线样例

```
时间线:
09:30:00 - 数据A: main_net=+100万, cumulative=+100万 (首次)
09:30:03 - 数据B: main_net=+50万, cumulative=+150万
09:30:15 - 无数据
09:30:30 - 数据C

【09:30:30计算】

1. 找上一个有数据点: 09:30:15? 无 → 09:30:03 ✓

2. 加载df_prev_main (09:30:03数据):
   - stock_code: 000001
   - main_net_amount: +50万
   - cumulative_main_net: +150万

3. 计算当前main_net_amount (09:30:30 - 09:30:03):
   - delta_amount = amount_093030 - amount_093003
   - main_net_amount = +80万

4. 【直接使用df_prev_main】计算cumulative_main_net:
   - prev_cumulative = 150万 (来自df_prev_main)
   - current_main = 80万
   - new_cumulative = 150万 + 80万 = 230万

结果:
09:30:30数据: main_net=+80万, cumulative=+230万 ✓
```
