## 一、当前查询流程分析

### 1.1 涨停标签获取流程 (`get_ztb_tags`)

```
用户请求 /api/stock-picker/ztb-tags?date=20260423
    ↓
get_ztb_tags(date)
    ↓
【步骤1】判断日期类型
    ├── 当天 (date == today): 查询 monitor_gp_sssj_{date}
    └── 历史: 查询 analysis_ztb_detail_{year}
    ↓
【步骤2】获取涨停股票代码
    ├── SQL: SELECT DISTINCT stock_code FROM monitor_gp_sssj_{date} WHERE is_zt = 1
    └── 结果: zt_codes = ['000001', '000002', ...]
    ↓
【步骤3】从内存缓存获取行业和概念
    ├── 检查 _stock_cache 是否加载
    └── 遍历 zt_codes，从 _stock_cache 中获取 industries 和 concepts
    ↓
【步骤4】统计频次
    ├── 统计每个行业出现的次数
    └── 统计每个概念出现的次数
    ↓
【步骤5】排序并返回
    └── 按频次降序排列，返回 industries 和 concepts 列表
```

### 1.2 交叉选股查询流程 (`query_cross_stocks`)

```
用户选择标签后点击查询
    ↓
query_cross_stocks(selected_tags)
    ↓
【步骤1】检查内存缓存
    └── 如果 _stock_cache 为空，调用 load_memory_cache()
    ↓
【步骤2】内存中匹配股票
    ├── 遍历 _stock_cache 中所有股票 (约5000+只)
    ├── 检查每只股票的行业和概念是否与 selected_tags 匹配
    └── 收集匹配的股票列表
    ↓
【步骤3】查询实时价格
    ├── 调用 query_realtime_prices(all_codes)
    ├── SQL: SELECT stock_code, short_name, price, change_pct 
    │        FROM monitor_gp_sssj_{date} 
    │        WHERE time = (SELECT MAX(time) FROM monitor_gp_sssj_{date})
    └── 结果: price_data
    ↓
【步骤4】查询转债实时价格
    ├── 调用 query_bond_realtime_prices(bond_codes)
    ├── SQL: SELECT bond_code, price, change_pct 
    │        FROM monitor_zq_sssj_{date}
    │        WHERE time = (SELECT MAX(time) FROM monitor_zq_sssj_{date})
    └── 结果: bond_price_data
    ↓
【步骤5】组装结果并分组
    ├── 按命中标签数量分组 (命中6个、5个...)
    └── 每组内按涨跌幅排序
    ↓
【步骤6】返回结果
    └── 返回 JSON 给前端
```

## 二、性能瓶颈分析

### 2.1 当前流程问题

| 问题点 | 说明 | 影响 |
|--------|------|------|
| **没有使用Redis** | 所有查询直接走MySQL | 高 |
| **数据库缺少索引** | `monitor_gp_sssj_{date}.is_zt` 无索引 | 高 |
| **子查询效率低** | `MAX(time)` 子查询每次执行 | 中 |
| **内存遍历O(n)** | 遍历所有5000+只股票 | 中 |
| **IN查询性能差** | `stock_code IN (...)` 列表长 | 中 |

### 2.2 具体SQL分析

#### 问题SQL 1: 涨停股票查询
```sql
SELECT DISTINCT stock_code FROM monitor_gp_sssj_{date} WHERE is_zt = 1
```
**问题:**
- `is_zt` 字段无索引，全表扫描
- `DISTINCT` 需要排序去重

**优化:**
```sql
-- 添加索引
ALTER TABLE monitor_gp_sssj_{date} ADD INDEX idx_is_zt (is_zt);

-- 或使用覆盖索引
ALTER TABLE monitor_gp_sssj_{date} ADD INDEX idx_is_zt_code (is_zt, stock_code);
```

#### 问题SQL 2: 实时价格查询
```sql
SELECT stock_code, short_name, price, change_pct 
FROM monitor_gp_sssj_{date} 
WHERE time = (SELECT MAX(time) FROM monitor_gp_sssj_{date}) 
  AND stock_code IN ('000001', '000002', ...)
```
**问题:**
- 子查询 `(SELECT MAX(time) ...)` 每次执行
- `stock_code IN (...)` 列表可能很长（100+只）
- `time` 字段无索引

**优化:**
```sql
-- 先获取最大时间（缓存）
SET @max_time = (SELECT MAX(time) FROM monitor_gp_sssj_{date});

-- 再查询价格
SELECT stock_code, short_name, price, change_pct 
FROM monitor_gp_sssj_{date} 
WHERE time = @max_time AND stock_code IN (...);

-- 添加索引
ALTER TABLE monitor_gp_sssj_{date} ADD INDEX idx_time (time);
ALTER TABLE monitor_gp_sssj_{date} ADD INDEX idx_time_code (time, stock_code);
```

### 方案B: 使用Redis缓存涨停数据

#### 3.2.1 缓存设计

```python
# Redis Key设计
ztb:codes:{date}           # 涨停股票代码集合 (Set)
ztb:tags:{date}:industries # 行业标签频次 (Sorted Set)
ztb:tags:{date}:concepts   # 概念标签频次 (Sorted Set)
ztb:timestamp:{date}       # 数据更新时间
```

#### 3.2.2 代码修改

```python
def get_ztb_tags(date: str = None) -> dict:
    """获取涨停行业概念标签（带Redis缓存）"""
    if not date:
        date = datetime.now().strftime('%Y%m%d')
    
    # 1. 尝试从Redis获取
    redis_client = get_redis_client()
    cache_key_codes = f"ztb:codes:{date}"
    cache_key_industries = f"ztb:tags:{date}:industries"
    cache_key_concepts = f"ztb:tags:{date}:concepts"
    
    cached_codes = redis_client.smembers(cache_key_codes)
    if cached_codes:
        logger.info(f"从Redis获取涨停数据: {len(cached_codes)} 只")
        zt_codes = list(cached_codes)
        # 从Redis获取标签统计
        industries = redis_client.zrevrange(cache_key_industries, 0, -1, withscores=True)
        concepts = redis_client.zrevrange(cache_key_concepts, 0, -1, withscores=True)
        return {
            'date': date,
            'total_zt': len(zt_codes),
            'industries': [{'name': name, 'type': 'industry', 'code': get_code(name), 'count': int(count)} 
                          for name, count in industries],
            'concepts': [{'name': name, 'type': 'concept', 'code': get_code(name), 'count': int(count)} 
                        for name, count in concepts]
        }
    
    # 2. 从MySQL查询（原逻辑）
    ...
    
    # 3. 写入Redis缓存（设置过期时间5分钟）
    pipe = redis_client.pipeline()
    pipe.sadd(cache_key_codes, *zt_codes)
    pipe.expire(cache_key_codes, 300)
    
    for ind in industries:
        pipe.zadd(cache_key_industries, {ind['name']: ind['count']})
    pipe.expire(cache_key_industries, 300)
    
    for con in concepts:
        pipe.zadd(cache_key_concepts, {con['name']: con['count']})
    pipe.expire(cache_key_concepts, 300)
    
    pipe.execute()
    
    return result
```

**预期效果:**
- 首次查询: 与MySQL查询相当
- 后续查询: 毫秒级（Redis内存操作）

### 方案C: 优化内存遍历（建立反向索引）

#### 3.3.1 反向索引设计

```python
# 在 stock_picker_service.py 中增加
_industry_to_stocks: Dict[str, Set[str]] = {}  # 行业 -> 股票代码集合
_concept_to_stocks: Dict[str, Set[str]] = {}   # 概念 -> 股票代码集合

def build_reverse_index():
    """构建反向索引"""
    global _industry_to_stocks, _concept_to_stocks
    
    _industry_to_stocks.clear()
    _concept_to_stocks.clear()
    
    for stock_code, data in _stock_cache.items():
        for industry in data['industries']:
            if industry not in _industry_to_stocks:
                _industry_to_stocks[industry] = set()
            _industry_to_stocks[industry].add(stock_code)
        
        for concept in data['concepts']:
            if concept not in _concept_to_stocks:
                _concept_to_stocks[concept] = set()
            _concept_to_stocks[concept].add(stock_code)
```

### 方案D: 缓存时间戳（避免重复子查询）

```python
# 全局缓存
_max_time_cache: Dict[str, str] = {}

def get_max_time(date: str, table_type: str = 'stock') -> str:
    """获取最大时间戳（带缓存）"""
    cache_key = f"{table_type}:{date}"
    
    # 检查缓存（5秒过期）
    if cache_key in _max_time_cache:
        cached_time, cached_at = _max_time_cache[cache_key]
        if time.time() - cached_at < 5:  # 5秒内有效
            return cached_time
    
    # 查询数据库
    mysql_tool = mysql_util.get_mysql_tool()
    table = f"monitor_gp_sssj_{date}" if table_type == 'stock' else f"monitor_zq_sssj_{date}"
    
    with mysql_tool.engine.connect() as conn:
        result = conn.execute(text(f"SELECT MAX(time) as max_time FROM {table}")).fetchone()
        max_time = result[0] if result else None
    
    # 写入缓存
    if max_time:
        _max_time_cache[cache_key] = (max_time, time.time())
    
    return max_time
```

## 六、当前流程确认

### 6.1 是否使用Redis?

**答案: 否**

当前 `get_ztb_tags` 和 `query_cross_stocks` 均未使用Redis：
- 涨停股票代码: 直接查询MySQL
- 实时价格: 直接查询MySQL
- 只有 `_stock_cache` 内存缓存（宽表数据）

### 6.2 查询路径

```
get_ztb_tags:
    MySQL (monitor_gp_sssj_{date} 或 analysis_ztb_detail_{year})
    ↓
    内存缓存 (_stock_cache)
    ↓
    返回结果

query_cross_stocks:
    内存缓存 (_stock_cache) - 匹配股票
    ↓
    MySQL (monitor_gp_sssj_{date}) - 实时价格
    ↓
    MySQL (monitor_zq_sssj_{date}) - 转债价格
    ↓
    返回结果
```

## 七、总结

### 主要问题
1. **数据库缺少索引** - 导致全表扫描
2. **未使用Redis缓存** - 每次查询都走MySQL
3. **子查询重复执行** - MAX(time) 每次查询
4. **内存遍历效率低** - O(n) 遍历所有股票

### 推荐方案
**立即实施: 方案A（添加索引）+ 方案D（缓存时间戳）**

预期效果:
- 查询耗时从秒级降至毫秒级
- 用户体验显著提升
